Custom Android back button behavior

Por padrão, quando o usuário pressiona o botão de voltar do hardware Android, react-navigation irá aparecer uma tela ou sair do aplicativo se não houver telas para mostrar. Esse é um comportamento padrão sensato, mas há situações em que você pode querer implementar o tratamento personalizado.

Temos, por exemplo, uma tela em que o usuário está selecionando itens em uma lista e um "modo de seleção" está ativo. Em um botão de voltar, pressione primeiro para que o "modo de seleção" seja desativado, e a tela deve ser exibida apenas ao pressionar o botão voltar novamente. O seguinte trecho de código demostra essa situação. Fazemos uso do BackHandler que vem com o react-native para adicionar nosso listener personalizado hardwareBackPress.

Retornando true do onBackButtonPressAndroid denota que nós manipulamos o evento, e o lister do react-navigation não será chamado, assim não estará aparecendo na tela. Retornando false fará com que o evento borbulhe e o listener do react-navigation irá aparecer na tela.

class ScreenWithCustomBackBehavior extends React.Component {
  componentDidMount() {
    BackHandler.addEventListener('hardwareBackPress', this.onBackButtonPressAndroid);
  }

  componentWillUnmount() {
    BackHandler.removeEventListener('hardwareBackPress', this.onBackButtonPressAndroid);
  }

  onBackButtonPressAndroid = () => {
    if (this.isSelectionModeEnabled()) {
      this.disableSelectionMode();
      return true;
    } else {
      return false;
    }
  };
}

A abordagem apresentada funcionará bem para telas mostradas em um StackNavigator. Personalizar o botão voltar em outras situações de manipulação pode não ser suportada no momento (por exemplo. Um caso conhecido quando isso não funciona é quando você deseja manipular o pressionamento do botão Voltar em um drawer aberto. Os PRs para tais casos de uso são bem-vindos!)
