Header buttons

Agora que sabemos como personalizar a aparência de nossos cabeçalhos, vamos torná-los sensiveis! Na verdade, talvez isso seja ambicioso, vamos apenas torná-los capazes de responder aos nossos toques de maneiras bem definidas.

Adicionando um botão para o cabeçalho

A maneira mais comum de interagir com um cabeçalho é tocando à esquerda ou à direita do título. Vamos adicionar um botão ao lado direito do cabeçalho (um dos lugares mais difíceis de tocar em sua tela inteira, dependendo do tamanho do dedo e do telefone, mas também um lugar normal para colocar botões).

class HomeScreen extends React.Component {
  static navigationOptions = {
    headerTitle: <LogoTitle />,
    headerRight: (
      <Button
        onPress={() => alert('This is a button!')}
        title="Info"
        color="#fff"
      />
    ),
  };
}

→ Executar esse código

A ligação de this em navigationOptions é * não * a instância HomeScreen, então você não pode chamar setState ou qualquer método de instância nele. Isso é muito importante porque é extremamente comum querer que os botões no cabeçalho interajam com a tela à qual o cabeçalho pertence. Então, vamos ver como fazer isso.

Existe um pacote desenvolvido pela comunidade para renderizar botões no cabeçalho com o estilo correto disponível react-navigation-header-buttons.

Interação de cabeçalho com seu componente de tela

O padrão mais comumente utilizado para dar acesso a um botão de cabeçalho a uma função na instância do componente é usar params. Vamos demonstrar isso com um exemplo clássico, o contador.

class HomeScreen extends React.Component {
  static navigationOptions = ({ navigation }) => {
    const params = navigation.state.params || {};

    return {
      headerTitle: <LogoTitle />,
      headerRight: (
        <Button onPress={params.increaseCount} title="+1" color="#fff" />
      ),
    };
  };

  componentWillMount() {
    this.props.navigation.setParams({ increaseCount: this._increaseCount });
  }

  state = {
    count: 0,
  };

  _increaseCount = () => {
    this.setState({ count: this.state.count + 1 });
  };

  /* later in the render function we display the count */
}

→ Executar esse código

O React Navigation não garante que o componente da tela comece a ser montado antes que o cabeçalho da tela seja renderizado e, como o parâmetro increaseCount está definido no componentWillMount, talvez não tenhamos está disponível para nós em navigationOptions, e é por isso que incluímos o || {} ao pegar os parâmetros (pode não haver nenhum). Sabemos que esta é uma API inábil e planejamos melhorá-la!

Como uma alternativa para setParams, você poderia usar uma biblioteca de gerenciamento de estado (como Redux ou MobX) e se comunicar entre o cabeçalho e a tela da mesma maneira que faria com dois componentes distintos.

Personalizando o botão Voltar

StackNavigator fornece os padrões específicos da plataforma para o botão Voltar. No iOS, isso inclui um rótulo ao lado do botão, que mostra o título da tela anterior quando o título cabe no espaço disponível, caso contrário, diz "Back".

Você pode alterar o comportamento de rótulo com headerBackTitle e headerTruncatedBackTitle (Leia mais).

Para personalizar a imagem do botão voltar, você pode usar headerBackImage.

Substituindo o botão Voltar

O botão voltar será processado automaticamente em um StackNavigator sempre que for possível para o usuário voltar de sua tela atual — Em outras palavras, o botão Voltar será renderizado sempre que houver mais de uma tela no stack.

Geralmente, isso é o que você quer. Mas é possível que em algumas circunstâncias você queira personalizar o botão Voltar mais do que você pode através das opções mencionadas acima, caso em que você pode especificar um headerLeft, como fizemos com o headerRight e substitua completamente o botão Voltar.

Sumário

  • Você pode definir botões no cabeçalho através das propriedades headerLeft e headerRight em navigationOptions.
  • O botão voltar é totalmente personalizável com headerLeft, mas se você quiser apenas alterar o título ou a imagem, existem outras navigationOptions para esse — headerBackTitle, headerTruncatedBackTitle e headerBackImage.
