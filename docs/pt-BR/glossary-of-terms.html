Glossário de termos

Esta é uma nova seção da documentação e falta muitos termos! Por favor, envie uma solicitação ou um problema com um termo que você acha que deve ser explicado aqui.

Header

Também conhecido como cabeçalho de navegação, barra de navegação, navbar e provavelmente muitas outras coisas. Este é o retângulo na parte superior da tela que contém o botão Voltar e o título da tela. O retângulo inteiro é geralmente chamado de cabeçalho no React Navigation.

Screen component

Um componente de tela é um componente que usamos em nossa configuração de rota.

const AppNavigator = createStackNavigator(
  {
    Home: {
      screen: HomeScreen,    // <----
    },
    Details: {
      screen: DetailsScreen, // <----
    },
  },
  {
    initialRouteName: 'Home',
  }
);

O sufixo Screen no nome do componente é inteiramente opcional, mas é uma convenção freqüentemente usada; nós poderíamos chamá-lo de CasaPantalla e isso funcionaria da mesma forma.

Vimos anteriormente que nossos componentes de tela são fornecidos com o prop navigation. É importante notar que isto só acontece se a tela é renderizada como uma rota por React Navigation (por exemplo, em resposta a this.props.navigation.navigate). Por exemplo, se renderizarmos DetailsScreen como um filho de HomeScreen, então DetailsScreen não será fornecido com o navigation prop e quando você pressionar o botão "Ir para detalhes... novamente "botão na tela inicial, o aplicativo irá lançar uma das excepções JavaScript" undefined não é um objeto ".

class HomeScreen extends React.Component {
  render() {
    return (
      <View style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}>
        <Text>Home Screen</Text>
        <Button
          title="Go to Details"
          onPress={() => this.props.navigation.navigate('Details')}
        />
        <DetailsScreen />
      </View>
    );
  }
}

Navigation Prop

Este prop será transmitido para todas as telas e pode ser usado para o seguinte:

  • dispatch enviará uma ação até o roteador
  • state é a rota atual para a tela
  • getParam é um ajudante para acessar um parâmetro que pode estar na rota
  • navigate, goBack, etc estão disponíveis para despachar ações de maneira conveniente

Navegadores também podem aceitar um suporte de navegação, que eles devem obter do navegador pai, se houver um.

Para mais detalhes, veja o "Navigation prop document".

Navigation State

O estado de um navegador geralmente é algo como isto:

{
  key: 'StackRouterRoot',
  index: 1,
  routes: [
    { key: 'A', routeName: 'Home' },
    { key: 'B', routeName: 'Profile' },
  ]
}

Para este estado de navegação, há duas rotas (que podem ser tabs, ou cards em um stack). O índice indica a rota ativa, que é "B".

Route

Cada rota é um pedaço de estado de navegação que contém uma chave para identificar isso e um "routeName" para designar o tipo de rota. Também pode conter parâmetros arbitrários:

{
  key: 'B',
  routeName: 'Profile',
  params: { id: '123' }
}

Child Navigation State

Ao compor navegadores, é possível que uma rota seja um estado de navegação. Ficaria assim:

{
  key: 'B',
  routeName: 'Profile',
  params: { id: '123' },
  index: 1,
  routes: [ {...}, {...} ]
}
