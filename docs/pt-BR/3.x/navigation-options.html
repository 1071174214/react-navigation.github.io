navigation-options

Cada tela pode configurar vários aspectos sobre como ela é apresentada nos navegadores principais. Você pode configurar

Configuração através do Static: Cada opção de navegação pode ser atribuída diretamente:

class MyScreen extends React.Component {
  static navigationOptions = {
    title: 'Great',
  };
  ...

Configuração Dinâmica

Ou, as opções podem ser uma função que recebe os seguintes argumentos e retorna um objeto de opções de navegação que substituirá as navigationOptions definidas pela rota e definidas pelo navegador.

  • props - Os mesmos "props" que estão disponíveis para o componente de tela
    • navigation - The navigation prop for the screen, with the screen's route at navigation.state
    • screenProps - Os suportes que passam acima do componente do navegador
    • navigationOptions - As opções padrão ou anterior que seriam usadas se novos valores não fossem fornecidos
class ProfileScreen extends React.Component {
  static navigationOptions = ({ navigation, screenProps }) => ({
    title: navigation.state.params.name + "'s Profile!",
    headerRight: <Button color={screenProps.tintColor} {...} />,
  });
  ...

Os screenProps são transmitidos em tempo de renderização. Se esta tela foi hospedada em um navegador SimpleApp:

<SimpleApp
  screenProps={{tintColor: 'blue'}}
  // navigation={{state, dispatch}} // Opção para controlar o App.
/>

Opções Genéricas de Navegação

A opção de navegação title é genérica entre todos os navegadores. É usado para definir a string de título para uma determinada tela.

class MyScreen extends React.Component {
  static navigationOptions = {
    title: 'Great',
  };
  ...

Ao contrário das outras opções de navegação, que são utilizadas apenas pela visualização do navegador, a opção de título pode ser usada pelo ambiente para atualizar o título na janela do navegador ou no alternador de aplicativos(App switcher).

Opções de Navegação Padrão

É muito comum definir navigationOptions em uma tela, mas às vezes é útil definir navigationOptions em um navegador também.

Imagine the following scenario: Your TabNavigator represents one of the screens in the app, and is nested within a top-level StackNavigator:

StackNavigator({
  route1: { screen: RouteOne },
  route2: { screen: MyTabNavigator },
});

Agora, quando route2 está ativo, você gostaria de mudar a cor de um cabeçalho. É fácil fazê-lo para route1, e também deve ser fácil fazê-lo para route2. Isto é o que são as opções de navegação padrão - elas são simplesmente navigationOptions definidas em um navegador:

const MyTabNavigator = TabNavigator({
  profile: ProfileScreen,
  ...
}, {
  navigationOptions: {
    headerTintColor: 'blue',
  },
});

Observe que você ainda pode decidir também especificar as navigationOptions nas telas das extremidades da hierarchia - por exemplo, o ProfileScreen acima. As navigationOptions da tela serão mescladas tecla-a-chave com as opções padrão vindas do navegador. Sempre que o navegador e a tela definem a mesma opção (por exemplo, headerTintColor), a tela vence. Portanto, você pode alterar a cor do tom quando o ProfileScreen estiver ativo, fazendo o seguinte:

class ProfileScreen extends React.Component {
  static navigationOptions = {
    headerTintColor: 'black',
  };
  ...
}
