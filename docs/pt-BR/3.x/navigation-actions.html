NavigationActions reference

Todas as NavigationActions retornam um objeto que pode ser enviado ao roteador usando o método navigation.dispatch().

Observe que, se você desejar disparar ações do react-navigation, deverá usar os criadores de ações fornecidos nesta biblioteca.

The following actions are supported:

  • Navigate - Navegar a uma outra rota
  • Back - Retornar ao estado prévio
  • Set Params - Setar os "Params" de uma dada rota
  • Init - Usado para inicializar o estado inicial se o estado for undefined

For actions specific to a StackNavigator, see StackActions.

The action creator functions define toString() to return the action type, which enables easy usage with third-party Redux libraries, including redux-actions and redux-saga.

navigate

The navigate action will update the current state with the result of a navigate action.

  • routeName - String - Obrigatório - Um routeName de destino que foi registrado em algum lugar no roteador do aplicativo
  • params - Object - Opicional - Params para combinar com os da rota de destino
  • action - Object - Opicional - (avançado) A sub-ação para executar no roteador filho, se a tela for um navegador. Qualquer uma das ações descritas neste documento pode ser definida como uma sub-ação.
  • key - String - Opcional - O identificador da rota para a qual navegar. Volte a esta rota se já existir
mport { NavigationActions } from 'react-navigation';

const navigateAction = NavigationActions.navigate({
  routeName: 'Profile',

  params: {},

  action: NavigationActions.navigate({ routeName: 'SubProfileRoute' }),
});

this.props.navigation.dispatch(navigateAction);

back

Go back to previous screen and close current screen. back action creator takes in one optional parameter:

  • key - string or null - opcional - Se definido, a navegação retornará da chave especificada. Se nulo, a navegação retornará a qualquer lugar.
import { NavigationActions } from 'react-navigation';

const backAction = NavigationActions.back({
  key: 'Profile',
});
this.props.navigation.dispatch(backAction);

setParams

When dispatching setParams, the router will produce a new state that has changed the params of a particular route, as identified by the key

  • params - object - obrigatório - Novos parâmetros a serem mesclados com os params de rota existentes
  • key - string - obrigatório - Chave de roteamento que deve receber os novos params
import { NavigationActions } from 'react-navigation';

const setParamsAction = NavigationActions.setParams({
  params: { title: 'Hello' },
  key: 'screen-123',
});
this.props.navigation.dispatch(setParamsAction);
