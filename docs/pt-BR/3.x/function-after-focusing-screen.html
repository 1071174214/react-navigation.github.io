Call a function when focused screen changes

Neste guia, vamos chamar uma ação na focagem da tela. Isso é útil para fazer chamadas de API adicionais quando um usuário revisita uma tela específica em um Tab Navigator ou para acompanhar os eventos do usuário ao tocar em nosso aplicativo.

Existem duas abordagens para chamar uma ação no foco da tela:

  1. Usando o withNavigationFocus maior ordem componente fornecido por react-navigation.
  2. Escutando o evento de'didFocus' com um ouvinte de evento.

Desencadear uma ação com o componente withNavigationFocus maior ordem

react-navigation fornece um componente de ordem superior que passa um isFocused prop ao nosso componente, juntamente com o navigation objeto que normalmente teria com withNavigation.

Quando o prop isFocused é passado para nosso componente, ele passará true quando a tela estiver focada e false quando nosso componente não estiver mais focado. Isso nos permite chamar ações em um usuário que entra ou sai de uma tela. Isso é particularmente útil quando tentamos interromper algo quando a página está desfocada, como interromper a reprodução de um arquivo de vídeo ou áudio ou interromper o rastreamento da localização de um usuário.

Uma vez que withNavigationFocus passa um prop em cada mudança de foco, fará com que nosso componente renderize novamente quando focalizarmos e desfocarmos uma tela. O uso desse componente de ordem superior pode introduzir re-renderizações de componentes desnecessárias à medida que a tela entra e sai de foco. Isso pode causar problemas, dependendo do tipo de ação que estamos chamando de foco.

Por exemplo, se estamos tentando fazer uma chamada de API em foco para buscar alguns dados, só queremos buscar dados quando o componente está focado e não quando o componente fica desfocado. Para evitar re-renderizações de componentes extras, poderíamos escrever alguma lógica em shouldComponentUpdate para controlar quando o componente se renderiza, no entanto, poderemos estar em melhor situação usando o método event listener detalhado abaixo. O ouvinte de evento só chamará uma ação e renderizará o componente quando a tela estiver focada e não fará nada quando uma tela ficar desfocada.

Example

import React, { Component } from "react";
import { View } from "react-native";
import { withNavigationFocus } from "react-navigation";

class TabScreen extends Component {
  componentDidUpdate(prevProps) {
    if (prevProps.isFocused !== this.props.isFocused) {
      // Use the `this.props.isFocused` boolean
      // Call any action
    }
  }

  render() {
    return <View />;
  }
}

// withNavigationFocus returns a component that wraps TabScreen and passes
// in the navigation prop
export default withNavigationFocus(TabScreen);

Este exemplo também está documentado no withNavigationFocus Documentação da API.

Acionando uma ação com um listener de evento 'didFocus'

Também podemos ouvir o evento 'didFocus' com um ouvinte de evento. Depois de configurar um ouvinte de eventos, devemos também parar de ouvir o evento quando a tela é desmontada.

Com essa abordagem, só poderemos chamar uma ação quando a tela estiver focada. Isso é ótimo para buscar dados com uma chamada de API quando uma tela fica focalizada ou qualquer outra ação que precisa acontecer quando a tela é exibida.

Example

import React, { Component } from "react";
import { View } from "react-native";
import { withNavigation } from "react-navigation";

class TabScreen extends Component {
  componentDidMount() {
    const { navigation } = this.props;
    this.focusListener = navigation.addListener("didFocus", () => {
      // The screen is focused
      // Call any action
    });
  }

  componentWillUnmount() {
    // Remove the event listener
    this.focusListener.remove();
  }

  render() {
    return <View />;
  }
}

export default withNavigation(TabScreen);
