Abrindo um modal em tela cheia

O Dictionary.com não fornece uma definição satisfatória de modal no que se refere às interfaces de usuário, mas a UI semântica a descreve da seguinte forma:

Um modal exibe conteúdo que bloqueia temporariamente as interações com a exibição principal

Isso parece certo. Um modal é como um popup — não faz parte do seu fluxo de navegação principal — normalmente, ela tem uma transição diferente, uma maneira diferente de descartá-la e destina-se a se concentrar em uma parte específica de conteúdo ou interação.

O propósito de explicar isso como parte dos fundamentos do React Navigation não é apenas porque este é um caso de uso comum, mas também porque a implementação requer conhecimento de nesting navigators, que é uma parte importante do React Navigation.

Criando um modal stack

class HomeScreen extends React.Component {
  static navigationOptions = ({ navigation }) => {
    const params = navigation.state.params || {};

    return {
      headerLeft: (
        <Button
          onPress={() => navigation.navigate('MyModal')}
          title="Info"
          color="#fff"
        />
      ),
      /* the rest of this config is unchanged */
    };
  };

  /* render function, etc */
}

class ModalScreen extends React.Component {
  render() {
    return (
      <View style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}>
        <Text style={{ fontSize: 30 }}>This is a modal!</Text>
        <Button
          onPress={() => this.props.navigation.goBack()}
          title="Dismiss"
        />
      </View>
    );
  }
}

const MainStack = createStackNavigator(
  {
    Home: {
      screen: HomeScreen,
    },
    Details: {
      screen: DetailsScreen,
    },
  },
  {
    /* Same configuration as before */
  }
);

const RootStack = createStackNavigator(
  {
    Main: {
      screen: MainStack,
    },
    MyModal: {
      screen: ModalScreen,
    },
  },
  {
    mode: 'modal',
    headerMode: 'none',
  }
);

→ Executar esse código

Existem algumas coisas importantes a notar aqui:

  • Como sabemos, a função stack navigator retorna um componente React (lembre-se que renderizamos < RootStack /> em nosso componente App). Este mesmo componente pode ser usado como um componente de tela! Ao fazer isso, estamos aninhando um navegador stack dentro de outro navegador stack. Nesse caso, isso é útil para nós, porque queremos usar um estilo de transição diferente para o modal e queremos desabilitar o cabeçalho em todo o stack. No futuro, isso será importante porque, para a navegação por abas, por exemplo, cada aba provavelmente terá seu próprio stack! Intuitivamente, isso é o que você espera: quando estiver na guia A e alternar para a guia B, você desejará que a guia A mantenha seu estado de navegação enquanto continua explorando a guia B. Veja este diagrama para visualizar a estrutura de navegação neste exemplo.

  • A configuração do mode para stack navigator pode ser card (padrão) ou modal. O comportamento modal desliza a tela de baixo para cima no iOS e permite que o usuário deslize de cima para baixo para dispensá-lo. A configuração do modal não tem efeito no Android porque os modais de tela cheia não têm nenhum comportamento de transição diferente na plataforma.

  • Quando chamamos navigate, não precisamos especificar nada, exceto a rota que gostaríamos de navegar. Não há necessidade de qualificar o stack que pertence (o stack 'root' ou 'principal' atribuída de forma arbitrária) — React Navigation tenta encontrar a rota no navegador mais próximo e, em seguida, executa a ação lá. Para visualizar isso, olhe novamente para este diagrama e imagine a ação navigate fluindo para cima de HomeScreen para MainStack, sabemos que o MainStack não pode manipular a rota MyModal, então ele flui para RootStack, que pode manipular essa rota e assim faz.

    Sumário

  • Para alterar o tipo de transição em um navegador stack, você pode usar a configuração mode. Quando definido como modal, todas as telas são animadas de baixo para cima e não da direita para a esquerda. Isso se aplica a todo esse navegador stack, portanto, para usar transições da direita para a esquerda em outras telas, adicionamos outro stack de navegação com a configuração padrão.

  • this.props.navigation.navigate percorre a árvore do navegador para encontrar um navegador que possa manipular a ação navigate.
