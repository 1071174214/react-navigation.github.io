createDrawerNavigator

createDrawerNavigator(RouteConfigs, DrawerNavigatorConfig);

RouteConfigs

O objeto configs da rota é um mapeamento do nome da rota para uma configuração da rota, que informa ao navegador o que apresentar para essa rota, consulte exemplo de createStackNavigator.

DrawerNavigatorConfig

  • drawerWidth - Largura do drawer ou função que retorna isso.
  • drawerPosition - opções são left ou right. Padrão da posição é left.
  • contentComponent - Componente usado para renderizar o conteúdo do drawer, por exemplo, itens de navegação. Recebe o navigation prop para o drawer. Padrão para DrawerItems. Para obter mais informações, consulte abaixo.
  • contentOptions - Configure o conteúdo do drawer, veja abaixo.
  • useNativeAnimations - Ativa animações nativas. Padrão é true.
  • drawerBackgroundColor - Use para alterar a cor de fundo do Drawer. O Padrão é white.
  • navigationOptions - Navigation options for the navigator itself, to configure a parent navigator
  • defaultNavigationOptions - Default navigation options to use for screens

The DrawerNavigator uses DrawerLayout under the hood, therefore it inherits the following props:

  • drawerType - One of front | back | slide
  • edgeWidth - Allows for defining how far from the edge of the content view the swipe gesture should activate
  • hideStatusBar - when set to true Drawer component will hide the OS status bar whenever the drawer is pulled or when it's in an "open" state.
  • overlayColor - Color overlay to be displayed on top of the content view when drawer gets open. A solid color should be used as the opacity is added by the Drawer itself and the opacity of the overlay is animated (from 0% to 70%).

Várias opções são passadas para o roteador subjacente para modificar a lógica de navegação:

  • initialRouteName - O nome da rota inicial.
  • order - Array de nomes das rotas que define a ordem dos itens do drawer.
  • paths - Forneça um mapeamento de routeName para path config, que substitui os caminhos definidos no routeConfigs.
  • backBehavior - O botão voltar deve mudar para a rota inicial? Se sim, defina como initialRoute, caso contrário none. O comportamento padrão é initialRoute.

Fornecendo um contentComponent personalizado

O componente padrão do drawer é rolável e contém apenas links para as rotas no RouteConfig. Você pode facilmente substituir o componente padrão para adicionar um cabeçalho, rodapé ou outro conteúdo ao drawer. Por padrão, o drawer é rolável e suporta a área segura do iPhone X. Se você personalizar o conteúdo, não se esqueça de incluir o conteúdo em um SafeAreaView:

import { DrawerItems, SafeAreaView } from 'react-navigation';

const CustomDrawerContentComponent = props => (
  <ScrollView>
    <SafeAreaView style={styles.container} forceInset={{ top: 'always', horizontal: 'never' }}>
      <DrawerItems {...props} />
    </SafeAreaView>
  </ScrollView>
);

const styles = StyleSheet.create({
  container: {
    flex: 1,
  },
});

contentComponent also received a prop called drawerOpenProgress which is an animated value that represents the animated position of the drawer (0 is closed; 1 is open). This allows you to do interesting animations in your contentComponent, such as parallax motion of the drawer contents:

const CustomDrawerContentComponent = props => {
  const translateX = props.drawerOpenProgress.interpolate({
    inputRange: [0, 1],
    outputRange: [-100, 0],
  });

  return <Animated.View style={{ transform: [{ translateX }] }}>{/* ... drawer contents */}</Animated.View>;
};

contentOptions para DrawerItems

  • items - a matriz de rotas, pode ser modificada ou substituída
  • activeItemKey - chave identificando a rota ativa
  • activeTintColor - etiqueta e cor do ícone do rótulo ativo
  • activeBackgroundColor - cor de fundo do rótulo ativo
  • inactiveTintColor - etiqueta e cor do ícone do rótulo inativo
  • inactiveBackgroundColor - cor de fundo do rótulo inativo
  • onItemPress(route) - função a ser invocada quando um item é pressionado
  • itemsContainerStyle - objeto de estilo para a seção de conteúdo
  • itemStyle - objeto de estilo para o item, que pode conter um ícone e/ou um rótulo
  • labelStyle - objeto de estilo para sobrescrever o estilo Text dentro da seção de conteúdo, quando seu rótulo é uma string
  • activeLabelStyle - objeto de estilo para sobrescrever o estilo Text do rótulo ativo, quando seu rótulo é uma string (mesclada com labelStyle)
  • inactiveLabelStyle - objeto de estilo para sobrescrever o estilo Text do rótulo inativo, quando seu rótulo é uma string (mesclada com labelStyle)
  • iconContainerStyle - objeto de estilo para sobrescrever estilos de container dos ícones da View.

Exemplo:

contentOptions: {
  activeTintColor: '#e91e63',
  itemsContainerStyle: {
    marginVertical: 0,
  },
  iconContainerStyle: {
    opacity: 1
  }
}

Screen Navigation Options

title

Título genérico que pode ser usado como fallback para headerTitle e drawerLabel

drawerLabel

String, React Element or a function that given { focused: boolean, tintColor: string } returns a React.Node, to display in drawer sidebar. When undefined, scene title is used

drawerIcon

React Element ou uma função, que dado { focused: boolean, tintColor: string } retorna um React.Node, para exibir na barra lateral do drawer

drawerLockMode

Especifica o modo bloqueio do drawer. Isto também pode atualizar dinamicamente usando screenProps.drawerLockMode no seu roteador de nível superior.

Nesting drawer navigators inside others

Se um navegador drawer estiver aninhado em outro navegador que forneça alguma interface do usuário, por exemplo, um navegador tabs ou um navegador stack, a gaveta será renderizada abaixo da interface do usuário a partir desses navegadores. O drawer aparecerá abaixo do tab e abaixo do cabeçalho do stack. Você precisará tornar o navegador drawer o pai de qualquer navegador onde o drawer deve ser renderizado na parte superior de sua interface do usuário.
