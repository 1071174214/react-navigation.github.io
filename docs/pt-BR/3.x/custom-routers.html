Custom routers

Você pode criar seu próprio roteador construindo um objeto com as seguintes funções:

const MyRouter = {
  getStateForAction: (action, state) => ({}),
  getActionForPathAndParams: (path, params) => null,
  getPathAndParamsForState: (state) => null,
  getComponentForState: (state) => MyScreen,
  getComponentForRouteName: (routeName) => MyScreen,
};

// Now, you can make a navigator by putting the router on it:
class MyNavigator extends React.Component {
  static router = MyRouter;
  render() {
    ...
  }
}

getStateForAction(action, state)

Define o estado de navegação em resposta a uma determinada ação. Esta função será executada quando uma ação for passada para props.navigation.dispatch(, ou quando qualquer uma das funções auxiliares for chamada, como navigation.navigate(.

Normalmente, isso deve retornar um estado de navegação, com o seguinte formato:

{
  index: 1, // identifies which route in the routes array is active
  routes: [
    {
      // Each route needs a name to identify the type.
      routeName: 'MyRouteName',

      // A unique identifier for this route in the routes array:
      key: 'myroute-123',
      // (used to specify the re-ordering of routes)

      // Routes can have any data, as long as key and routeName are correct
      ...randomRouteData,
    },
    ...moreRoutes,
  ]
}

Se o roteador tiver manipulado a ação externamente sem alterar o estado de navegação, essa função retornará null.

getComponentForRouteName(routeName)

Retorna o componente filho ou navegador para o nome de rota fornecido.

Digamos que um roteador getStateForAction produza um estado como este:

{
  index: 1,
  routes: [
    { key: 'A', routeName: 'Foo' },
    { key: 'B', routeName: 'Bar' },
  ],
}

Baseado no routeNames no estado, o roteador é responsável por retornar componentes válidos ao chamar router.getComponentForRouteName ('Foo') ou router.getComponentForRouteName ('Bar').

getComponentForState(state)

Retorna o componente ativo para um estado de navegação profunda.

getActionForPathAndParams(path, params)

Retorna uma ação de navegação opcional que deve ser usada quando o usuário navega para esse caminho e fornece parâmetros de consulta opcionais.

getPathAndParamsForState(state)

Retorna o caminho e os parâmetros que podem ser colocados no URL para vincular o usuário ao mesmo local no aplicativo.

O caminho/parâmetros que são gerados a partir disso devem formar uma ação quando passados de volta para o getActionForPathAndParams do roteador. Essa ação deve levá-lo a um estado semelhante depois de passar por getStateForAction.

getScreenOptions(navigation, screenProps)

Usado para recuperar as opções de navegação para uma tela. Deve fornecer suporte de navegação atual da tela e, opcionalmente, outros adereços que suas opções de navegação podem precisar consumir.

  • navigation - Este é o navigation prop que a tela usará, onde o estado se refere à rota/estado da tela. Despacho irá desencadear ações no contexto dessa tela.
  • screenProps - Outros props que suas opções de navegação podem precisar consumir
  • navigationOptions - O conjunto anterior de opções que são padrão ou fornecidas pelo configurador anterior

Dentro de uma View de exemplo, talvez você precise buscar o título configurado:

// Primeiro, prepare o navigation prop para o seu filho ou reutilize um, se já estiver disponível.
const screenNavigation = addNavigationHelpers({
  // In this case we use navigation.state.index because we want the title for the active route.
  state: navigation.state.routes[navigation.state.index],
  dispatch: navigation.dispatch,
});
const options = this.props.router.getScreenOptions(screenNavigation, {});
const title = options.title;
