Configurando a barra de cabeçalho

Até agora você provavelmente está cansado de ver uma barra cinza em branco na parte superior da tela — você está pronto para algum flair. Então, vamos entrar na configuração da barra de cabeçalho.

Configurando o título do cabeçalho

Um componente de tela pode ter uma propriedade estática chamada navigationOptions, que é um objeto ou uma função que retorna um objeto que contém várias opções de configuração. O que usamos para o título do cabeçalho é title, como demonstrado no exemplo a seguir.

class HomeScreen extends React.Component {
  static navigationOptions = {
    title: 'Home',
  };

  /* render function, etc */
}

class DetailsScreen extends React.Component {
  static navigationOptions = {
    title: 'Details',
  };

  /* render function, etc */
}

→ Executar esse código

createStackNavigator usa convenções de plataforma por padrão, então no iOS o título será centralizado e no Android ele será alinhado à esquerda.

Usando params no título

Para usar params no título, precisamos tornar navigationOptions uma função que retorna um objeto de configuração. Pode ser tentador tentar usar this.props dentro de navigationOptions, mas porque é uma propriedade estática do componente, isso não consulte uma instância do componente e, portanto, nenhum objeto está disponível. Ao invés disso, se nós fizermos navigationOptions uma função, então o React Navigation irá chamá-la com um objeto contendo { navigation, navigationOptions, screenProps } - nesse caso, tudo o que nos interessa é navigation, que é o mesmo objeto que é passado para sua tela como this.props.navigation. Você pode se lembrar que nós podemos pegar os parâmetros de navigation através de navigation.getParam ou navigation.state.params, e então fazemos isso abaixo para extraia um param e use-o como título.

class DetailsScreen extends React.Component {
  static navigationOptions = ({ navigation }) => {
    return {
      title: navigation.getParam('otherParam', 'A Nested Details Screen'),
    };
  };

  /* render function, etc */
}

→ Executar esse código

O argumento que é passado para a função navigationOptions é um objeto com as seguintes propriedades:

  • navigation - O navigation prop para a tela, com a rota da tela em navigation.state.
  • screenProps - Os suportes que passam acima do componente do navegador
  • navigationOptions - As opções padrão ou anterior que seriam usadas se novos valores não fossem fornecidos

Nós só precisamos do prop navigation no exemplo acima, mas você pode, em alguns casos, querer usar screenProps ou navigationOptions.

Atualizando navigationOptions com setParams

Geralmente, é necessário atualizar a configuração navigationOptions da tela ativa a partir do próprio componente de tela montado. Nós podemos fazer isso usando this.props.navigation.setParams

  /* Inside of render() */
  <Button
    title="Update the title"
    onPress={() => this.props.navigation.setParams({otherParam: 'Updated!'})}
  />

→ Executar esse código

Ajustar os estilos de cabeçalho

Existem três propriedades chaves para usar ao personalizar o estilo do seu cabeçalho: headerStyle, headerTintColor e headerTitleStyle.

  • headerStyle: um objeto de estilo que será aplicado ao View que envolve o cabeçalho. Se você definir backgroundColor nele, que será a cor do seu cabeçalho.
  • headerTintColor: o botão Voltar e o título usam essa propriedade como sua cor. No exemplo abaixo, definimos a cor do tom para branco (#fff) para que o botão Voltar e o título do cabeçalho sejam brancos.
  • headerTitleStyle: se quisermos personalizar o fontFamily, fontWeight e outro, podemos usar o Text e suas propriedades de estilo para isso.
class HomeScreen extends React.Component {
  static navigationOptions = {
    title: 'Home',
    headerStyle: {
      backgroundColor: '#f4511e',
    },
    headerTintColor: '#fff',
    headerTitleStyle: {
      fontWeight: 'bold',
    },
  };

  /* render function, etc */
}

→ Executar esse código

Há algumas coisas para notar aqui: 1. No iOS, o texto e os ícones da barra de status são pretos e isso não fica muito bem em um plano de fundo escuro. Não discutiremos isso aqui, mas você deve configurar a barra de status para se adequar às cores da tela conforme descrito no guia da barra de status. 2. A configuração que definimos só se aplica à tela inicial; Quando navegamos para a tela de detalhes, os estilos padrão estão de volta. Veremos como compartilhar navigationOptions entre as telas agora.

Compartilhando navigationOptions comuns em telas

É comum querer configurar o cabeçalho de maneira semelhante em várias telas. Por exemplo, a cor da marca da sua empresa pode ser vermelha e, portanto, você deseja que a cor do plano de fundo do cabeçalho seja vermelha e que a cor do tom seja branca. Convenientemente, estas são as cores que estamos usando em nosso exemplo de execução, e você notará que quando você navega para a DetailsScreen, as cores retornam aos padrões. Não seria horrível se tivéssemos que copiar as propriedades de estilo de cabeçalho navigationOptions de HomeScreen para DetailsScreen, e para cada componente de tela único que usamos em nosso aplicativo? Felizmente, não temos. Em vez disso, podemos mover a configuração para o navegador do stack sob a propriedade defaultNavigationOptions.

class HomeScreen extends React.Component {
  static navigationOptions = {
    title: 'Home',
    /* No more header config here! */
  };

  /* render function, etc */
}

/* other code... */

const RootStack = createStackNavigator(
  {
    Home: HomeScreen,
    Details: DetailsScreen,
  },
  {
    initialRouteName: 'Home',
    /* The header config from HomeScreen is now here */
    defaultNavigationOptions: {
      headerStyle: {
        backgroundColor: '#f4511e',
      },
      headerTintColor: '#fff',
      headerTitleStyle: {
        fontWeight: 'bold',
      },
    },
  }
);

→ Executar esse código

Agora, qualquer tela que pertence a RootStack terá nossos estilos da marca maravilhosos. Certamente, deve haver uma maneira de substituir essas opções, se precisarmos?

Nota: Em v2 e abaixo, a propriedade que você iria querer usar para fazer isso é navigationOptions. Em v3 já renomeamos este para defaultNavigationOptions.

A propriedade navigationOptions pode ser usada para configurar o próprio navegador:

const Home = createStackNavigator(
  {
    Feed: ExampleScreen,
    Profile: ExampleScreen,
  }, {
    defaultNavigationOptions: {
      headerTintColor: '#fff',
      headerStyle: {
        backgroundColor: '#000',
      },
    },
    navigationOptions: {
      tabBarLabel: 'Home!',
    },
  }
);

const Tabs = createBottomTabNavigator({ Home });

→ Executar esse código

Substituindo navigationOptions compartilhados

As navigationOptions especificadas no seu componente de tela são mescladas com as opções de navegação padrão de seu navegador stack pai, com as opções no componente de tela tendo precedência. Let's use this knowledge to invert the background and tint colors on the details screen.

class DetailsScreen extends React.Component {
  static navigationOptions = ({ navigation, navigationOptions }) => {
    const { params } = navigation.state;

    return {
      title: params ? params.otherParam : 'A Nested Details Screen',
      /* These values are used instead of the shared configuration! */
      headerStyle: {
        backgroundColor: navigationOptions.headerTintColor,
      },
      headerTintColor: navigationOptions.headerStyle.backgroundColor,
    };
  };

  /* render function, etc */
}

→ Executar esse código

Substituindo o título por um componente personalizado

Sometimes you need more control than just changing the text and styles of your title -- for example, you may want to render an image in place of the title, or make the title into a button. In these cases you can completely override the component used for the title and provide your own.

class LogoTitle extends React.Component {
  render() {
    return (
      <Image
        source={require('./spiro.png')}
        style={{ width: 30, height: 30 }}
      />
    );
  }
}

class HomeScreen extends React.Component {
  static navigationOptions = {
    // headerTitle instead of title
    headerTitle: <LogoTitle />,
  };

  /* render function, etc */
}

→ Executar esse código

You might be wondering, why headerTitle when we provide a component and not title, like before? The reason is that headerTitle is a property that is specific to a stack navigator, the headerTitle defaults to a Text component that displays the title.

Configuração adicional

You can read the full list of available navigationOptions for screens inside of a stack navigator in the createStackNavigator reference.

Sumário

  • Você pode personalizar o cabeçalho dentro da propriedade estática navigationOptions nos componentes da tela. Leia a lista completa de opções na referência da API.
  • A propriedade estática navigationOptions pode ser um objeto ou uma função. Quando é uma função, é fornecida com um objeto com o prop navigation, screenProps e navigationOptions nele.
  • Você também pode especificar navigationOptions compartilhados na configuração do navegador stack ao inicializá-lo. A propriedade estática tem precedência sobre essa configuração.
