Acesse o navigation prop de qualquer componente

withNavigation é um componente de ordem superior que passa a prop navigation para um Componente empacotado. É útil quando você não pode passar o navigation prop diretamente para o componente, ou não deseja passá-lo no caso de um filho profundamente aninhado.

import React from 'react';
import { Button } from 'react-native';

export default class MyBackButton extends React.Component {
  render() {
    // This will throw an 'undefined is not a function' exception because the navigation
    // prop is undefined.
    return <Button title="Back" onPress={() => { this.props.navigation.goBack() }} />;
  }
}

Para resolver esta exceção, você poderia passar o prop navigation para MyBackButton quando você o renderiza em uma tela, assim: <MyBackButton navigation={this.props.navigation} />.

Como alternativa, você pode usar a função withNavigation para fornecer o suporte de navegação automaticamente (através de React context, se você estiver curioso). Esta função se comporta de forma semelhante à função connect do Redux, exceto que em vez de fornecer o prop dispatch para o componente que ela envolve, ela fornece o prop navigation.

import React from 'react';
import { Button } from 'react-native';
import { withNavigation } from 'react-navigation';

class MyBackButton extends React.Component {
  render() {
    return <Button title="Back" onPress={() => { this.props.navigation.goBack() }} />;
  }
}

// withNavigation returns a component that wraps MyBackButton and passes in the
// navigation prop
export default withNavigation(MyBackButton);

Usando essa abordagem, você pode processar MyBackButton em qualquer lugar do seu aplicativo sem passar explicitamente em uma proposição de navegação e isso funcionará como esperado.
