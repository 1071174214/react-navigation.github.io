Olá React Navigation

Em um navegador da Web, você pode vincular a páginas diferentes usando uma tag âncora (<a>). Quando o usuário clica em um link, o URL é enviado para a pilha de histórico do navegador. Quando o usuário pressiona o botão Voltar, o navegador exibe o item da parte superior da pilha do histórico, portanto, a página ativa é agora a página visitada anteriormente. O React Native não tem uma ideia embutida de uma pilha de histórico global como um navegador da Web - é aqui que o React Navigation entra na história.

O navegador stack do React Navigation fornece uma maneira de o seu aplicativo fazer a transição entre as telas e gerenciar o histórico de navegação. Se o seu aplicativo usar apenas um navegador stack, ele será conceitualmente semelhante ao modo como um navegador da Web manipula o estado de navegação - seu aplicativo envia e empurra itens do stack de navegação conforme os usuários interagem e isso faz com que o usuário veja telas diferentes. Uma diferença fundamental entre como isso funciona em um navegador da Web e no React Navigation é que o navegador stack do React Navigation fornece os gestos e animações que você esperaria no Android e no iOS ao navegar entre rotas no stack.

Vamos começar demonstrando o navegador mais comum, createStackNavigator.

Criando um navegador stack

createStackNavigator é uma função que retorna um React component. Leva um objeto de configuração de rota e, opcionalmente, um objeto de opções (omitimos isso abaixo, por enquanto). createAppContainer is a function that retuns a React component to take as a parameter the React component created by the createStackNavigator, and can be directly exported from App.js to be used as our App's root component.

// In App.js in a new project

import React from "react";
import { View, Text } from "react-native";
import { createStackNavigator, createAppContainer } from "react-navigation";

class HomeScreen extends React.Component {
  render() {
    return (
      <View style={{ flex: 1, alignItems: "center", justifyContent: "center" }}>
        <Text>Home Screen</Text>
      </View>
    );
  }
}

const AppNavigator = createStackNavigator({
  Home: {
    screen: HomeScreen
  }
});

export default createAppContainer(AppNavigator);

→ Executar esse código

Se você executar este código, verá uma tela com uma barra de navegação vazia e uma área de conteúdo cinza contendo seu componente HomeScreen. Os estilos que você vê para a barra de navegação e a área de conteúdo são a configuração padrão para um navegador stack, aprenderemos como configurá-los mais tarde.

O nome da rota não importa - você pode usar home em letras minúsculas ou Home em letras maiúsculas, cabe a você. Nós preferimos capitalizar nossos nomes de rota.

A única configuração necessária para uma rota é o componente screen. Você pode ler mais sobre as outras opções disponíveis na referência do StackNavigator .

No React Native, o componente exportado do App.js é o ponto de entrada (ou componente raiz) do seu aplicativo - é o componente do qual todos os outros componentes desce. Geralmente é útil ter mais controle sobre o componente na raiz do seu aplicativo do que você obteria ao exportar o resultado de createAppContainer, então vamos exportar um componente que apenas renderiza nosso AppNavigator navegador stack.

const AppContainer = createAppContainer(AppNavigator);

export default class App extends React.Component {
  render() {
    return <AppContainer />;
  }
}

Forma abreviada de configuração de rota

Dado que a única configuração de rota que temos para Home é o componente de tela, não precisamos usar o formato de configuração { screen: HomeScreen }, podemos usar a tela componente diretamente.

const AppNavigator = createStackNavigator({
  Home: HomeScreen
});

Adicionando uma segunda rota

O componente <AppContainer/> não aceita props - toda a configuração é especificada no parâmetro options para o código AppNavigator, pela função createStackNavigator. Deixamos o options em branco, então ele apenas usa a configuração padrão. Para ver um exemplo do uso do objeto options, adicionaremos uma segunda tela ao navegador stack.

// Outros códigos para o HomeScreen aqui...

class DetailsScreen extends React.Component {
  render() {
    return (
      <View style={{ flex: 1, alignItems: "center", justifyContent: "center" }}>
        <Text>Details Screen</Text>
      </View>
    );
  }
}

const AppNavigator = createStackNavigator(
  {
    Home: HomeScreen,
    Details: DetailsScreen
  },
  {
    initialRouteName: "Home"
  }
);

// Other code for App component here...

Agora nossa stack tem duas rotas, uma rota Home e uma rota Detalhes. A rota Home corresponde ao componente HomeScreen, e a rota Details corresponde ao componente DetailsScreen. A rota inicial para o stack é a rota Home. A questão natural neste momento é: "como eu vou da rota Home para a rota de Details?". Isso é coberto na próxima seção.

Sumário

  • O React Native não possui uma API integrada para navegação como um navegador da web. O React Navigation fornece isso para você, juntamente com os gestos e animações do iOS e do Android para fazer a transição entre as telas.
  • createStackNavigator é uma função que usa um objeto de configuração de rota e um objeto de opções e retorna um componente React.
  • As chaves no objeto de configuração de rota são os nomes de rota e os valores são a configuração dessa rota. A única propriedade requerida na configuração é o screen (o componente a ser usado para a rota).
  • Para especificar qual é a rota inicial em um stack, forneça um initialRouteName no objeto de opções do stack.
