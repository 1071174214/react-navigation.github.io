Using the navigation key

O parâmetro key aparece repetidamente em diferentes funções de navegação. Vamos dar uma olhada em um resumo de seus casos de uso:

Uso com a função navigate

Se nenhuma chave for fornecida, o StackRouter se comportará da seguinte maneira:

  • se uma rota com o nome fornecido já existir no estado, StackRouter irá pular para a rota existente, junto com a configuração dos novos parâmetros.
  • se não existir tal rota, o StackRouter irá empurrá-lo para a "stack"

Se, no entanto, você quiser enviar várias instâncias da mesma rota, você pode fazer isso fornecendo um parâmetro key único toda vez que chamar navigate, ou você pode usar o Ação push disponível no StackRouter. Veja o RFC para mais informações.

Nota: o comportamento de navigate sem uma key é significativamente diferente na série 1.x de lançamentos. Leia mais sobre isso aqui.

Uso com reset

Ao resetar,key também é opcional e pode ser uma string ou null. Se definido, o navegador com a chave especificada será resetado. Se null, o navegador raiz será resetado. Você pode obter a chave do navegador de uma rota chamando this.props.navigation.dangerouslyGetParent().state.key. O motivo pelo qual a função é chamada de dangerouslyGetParent é avisar os desenvolvedores sobre o uso excessivo dela, por exemplo. obter pai dos pais e outros padrões difíceis de seguir.

Uso com replace

Com a ação de navegação replace, key é um parâmetro obrigatório usado para identificar a rota a ser substituída. Se você usar a função this.props.navigation.replace, substituiremos automaticamente a chave da rota atual.

Uso com goBack

Por favor, consulte o goBack docs para uma explicação detalhada.
