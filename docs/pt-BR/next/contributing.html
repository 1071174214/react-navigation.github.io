Guia de colaboradores do React Navigation

Quer ajudar a melhorar o React Navigation? Sua ajuda seria muito apreciada!

Aqui estão algumas maneiras de contribuir com o projeto:

  • Reportando bugs
  • Melhorando a documentação
  • Respondendo a issues
  • Correções de bugs
  • Sugestão de um recurso
  • Grandes Pull Requests

E aqui estão alguns recursos úteis para ajudar na introdução:

  • Modelo de problema
  • Modelo de Pull Request
  • Forking do Repositorio
  • Diretrizes de Revisão de Código
  • Execute o aplicativo de exemplo
  • Execute o site
  • Rode testes

Observe que usamos o Yarn nos exemplos abaixo, mas você pode usar o NPM.

Contribuindo

Reportando bugs

Você não pode escrever código sem escrever o bug ocasional. Especialmente como React Navigation está se movendo rapidamente, erros acontecem. Quando você acha que encontrou um, aqui está o que fazer:

  1. Pesquise os problemas existentes para o que você está vendo. Se você ver um, adicione um Leia os comentários e veja se você pode fornecer mais informações valiosas para o segmento
  2. Se não houver outros problemas como o seu, crie um novo. Certifique-se de seguir o modelo de problema .

Criar uma reprodução de alta qualidade é fundamental. Sem isso, provavelmente não poderemos consertar o bug e, em uma situação ideal, você descobrirá que não é realmente um bug da biblioteca, mas simplesmente feito incorretamente em seu projeto. Correção instantânea de bugs!

Melhorando a documentação

Qualquer projeto bem-sucedido precisa de documentação de qualidade e o React Navigation não é diferente.

Leia mais sobre a documentação no repositório react-navigation / website .

Respondendo a issues

Outra ótima maneira de contribuir para o React Navigation é respondendo a problemas. Talvez esteja respondendo a pergunta de alguém, apontando um pequeno erro no código, ou ajudando-os a montar uma reprodução. Se você estiver interessado em uma função mais ativa no React Navigation, comece respondendo aos problemas - não apenas é útil, mas demonstra seu comprometimento e conhecimento do código!

Correções de bugs

Encontre um bug, conserte-o, o dia todo você terá boa sorte! Como foi mencionado anteriormente, erros acontecem. Se você encontrar um bug, faça o seguinte:

  1. Verifique se já existe um pedido pull que resolve esse erro. Se isso acontecer, dê uma revisão e deixe seus comentários
  2. Se ainda não houver um pedido de pull, descubra a correção! Se for relativamente pequeno, vá em frente e corrija-o e envie um pedido de pull. Se houver um número decente de alterações, envie primeiro um problema para que possamos discuti-lo (consulte a seção Solicitações de grande volume )
  3. Se houver um problema relacionado a esse bug, deixe um comentário sobre ele, vinculando-o à sua solicitação de recebimento, para que os outros saibam que ele foi corrigido.

Confira a ajuda desejada e bom primeiro issue tags para ver onde você pode começar a ajudar!

Sugestão de um recurso

Há algo que você quer ver no React Navigation? Por favor, crie uma solicitação de recurso no Canny.

Grandes Pull Requests

Para quaisquer alterações que irão adicionar / remover / modificar vários arquivos no projeto (novos recursos ou correções de bugs) adiar a escrita de código imediatamente. Há algumas razões para isso

  1. Grandes pull requests demoram muito tempo para serem revisadas e às vezes é difícil entender o contexto
  2. Muitas vezes você pode não ter que fazer uma mudança tão grande quanto você espera

Com isso em mente, aqui está a sugestão

  1. Abra um problema e defina claramente o que você deseja realizar e como você pretende realizá-lo
  2. Discuta essa solução com a comunidade e mantenedores. Fornecer contexto, estabelecer casos de uso e descobrir o design
  3. Decidir sobre um plano de ação
  4. Escrever o código e enviar o PR
  5. Rever o PR. Isto pode levar algum tempo, mas, se você seguiu os passos acima, espero que não vai demorar muito tempo.

A razão pela qual queremos fazer isso é poupar tempo a todos. Talvez esse recurso já exista, mas não esteja documentado? Ou talvez não se encaixe na biblioteca. Independentemente disso, discutindo uma mudança importante na frente você está economizando seu tempo e outros tempo também.

Alterações na interface & Tipos

Se você alguma vez encontrar uma alteração na interface pública do projeto (a API), certifique-se de atualizar as definições de biblioteca correspondentes para o Flow. Esses "libdefs" especificam os tipos de nossa API para que os usuários da biblioteca possam verificar o seu código. Um exemplo de uma mudança de qualificação seria adicionar uma nova opção de navegação.

O libdef (localizado em flow/react-navigation.js) precisará ser atualizado de forma que a execução do flow na pasta examples/NavigationPlayground produza sem erros.

  1. Siga as instruções na seção Executar o aplicativo de exemplo para preparar o exemplo do NavigationPlayground e instale o flow na pasta local do node_modules/.bin do código.
  2. Execute flow para ver se há erros.
  3. Se nenhum erro ocorrer como resultado de uma alteração na API, isso indica que não temos nenhuma cobertura no projeto de exemplo NavigationPlayground para sua alteração na API. Este é frequentemente o caso - por exemplo, se você adicionar uma nova opção de navegação. Nesse caso, você deve adicionar um exemplo de uso de seu novo recurso ao NavigationPlayground para poder testar suas alterações no libdef e para que possamos manter seu recurso devidamente testado e digitado em perpetuidade.
  4. Uma vez que você está vendo erros, vá em frente e atualize o libdef (localizado em flow/react-navigation.js) para que não haja mais nenhum erro quando você executar o flow dentro de exemplos /NavigationPlayground.
  5. Inclua as alterações da libdef no PR para o seu novo recurso. Certifique-se de sinalizar para os mantenedores que seu PR tem uma alteração de libdef, para que quando a próxima versão da biblioteca for liberada, nós façamos o upload da libdef atualizada para o repositório flow-typed.

Informação

Modelo de problema

Antes de enviar um problema, confira o modelo de problema e segui-lo. Isso está em vigor para ajudar todos a entender melhor o problema que você está tendo e reduzir o tempo todo para obter as informações necessárias.

Sim, é preciso tempo e esforço para concluir o modelo de problema. Mas essa é a única maneira de fazer perguntas de alta qualidade que realmente obtenham respostas.

Você prefere levar 1 minuto para criar um relatório de problema incompleto e esperar meses para receber algum tipo de resposta? Ou prefere levar 20 minutos para preencher um relatório de problemas de alta qualidade, com todos os elementos necessários, e obter uma resposta em dias? É também uma coisa respeitosa para quem está disposto a reservar um tempo para rever o seu problema.

Modelo de Pull Request

Muito parecido com o modelo de problema, o pull request template estabelece instruções para garantir que sua solicitação de"pull request" seja revisada em tempo hábil. Certifique-se de examiná-lo antes de começar a escrever qualquer código.

Forking do Repositorio

  • Fork react-navigation no GitHub
  • Execute estes comandos no terminal para baixar localmente e instalá-lo:
git clone https://github.com/<USERNAME>/react-navigation.git
cd react-navigation
git remote add upstream https://github.com/react-community/react-navigation.git
yarn install

Diretrizes de Revisão de Código

Combine o estilo do restante da base de código. Este projeto usa o ESLint para garantir a consistência em todo o projeto. Você pode verificar seu projeto executando

yarn run eslint

Se ocorrer algum erro, você terá que corrigi-lo manualmente ou tentar corrigi-lo automaticamente executando o yarn run format.

Execute o aplicativo de exemplo

O aplicativo de exemplo NavigationPlayground inclui uma variedade de padrões e é usado de maneira simples para contribuidores para alterações de teste de integração manual. Veja as instruções do README para executá-lo.

Execute o site

Para o modo de desenvolvimento e live-reloading:

cd website
yarn install
yarn start

Para executar o site no modo de produção com a renderização do servidor:

yarn run prod

Se você fez alguma alteração no diretório docs você precisará executar yarn run build-docs a partir da raiz do projeto.

Rode testes

O React Navigation tem testes implementados em Jest. Para executar um desses, no diretório React Navigation, execute um dos seguintes comandos (após instalar o node_modules) para executar testes ou verificação de tipo.

yarn run jest

Estes comandos serão executados pelo nosso CI e são obrigados a passar antes que qualquer contribuição seja mesclada.
