Referência do Navigation prop

Cada componente screen no seu aplicativo é fornecido com o prop navigation automaticamente. O prop contém várias funções de conveniência que despacham ações de navegação no roteador da rota. Se parece com isso:

  • navigation
    • navigate - ir para outra tela, descobre a ação necessária para fazer isso
    • reset - limpe o estado do navegador e substitua-o pelo resultado de várias ações
    • goBack - feche a tela ativa e mover para trás na stack
    • addListener - registrar atualizações para o ciclo de vida da navegação
    • setParams - fazer alterações nos parâmetros da rota
    • dispatch - enviar uma ação ao roteador
    • dangerouslyGetParent - função que retorna o navegador pai, se houver

É importante destacar que a prop navigation não é passada para todos componentes; apenas componentes da tela recebem esta prop automaticamente! React Navigation doesn't do any magic here. Por exemplo, se você fosse definir um componente MyBackButton e renderizá-lo como um filho de um componente de tela, você não seria capaz de acessar a prop navigation nele. If, however, you wish to access the navigation prop in any of your components, you may use the useNavigation hook.

Funções dependentes do navegador

There are several additional functions present on navigation prop based on the kind of the current navigator.

Se o navegador for um stack navigator, várias alternativas para navigate e goBack serão fornecidas e você poderá usar o que preferir. As funções são:

  • navigation
    • replace - substitua a rota atual por uma nova
    • push - empurrar uma nova rota para a stack
    • pop - volte na stack
    • popToTop - vai para o topo da stack

If the navigator is a tab navigator, the following are also available:

  • navigation
    • jumpTo - go to a specific screen in the tab navigator

Se o navegador for um drawer navigator, os seguintes itens também estarão disponíveis:

  • navigation
    • jumpTo - go to a specific screen in the drawer navigator
    • openDrawer - abrir a drawer
    • closeDrawer - fechar a drawer
    • toggleDrawer - alternar o estado, ou seja. mudar de fechado para aberto e vice-versa

You can find more details about navigator dependent functions in the documentation for the navigator that you're using.

Referência de API comum

The vast majority of your interactions with the navigation prop will involve navigate, goBack, and setParams.

navigate - Link para outras telas

Chame isso para vincular a outra tela no seu aplicativo. Leva os seguintes argumentos:

navigation.navigate(name, params)

  • name - A destination name of the route that has been defined somewhere
  • params - Parâmetros para mesclar na rota de destino

function HomeScreen({ navigation: { navigate } }) {
  return (
    <View>
      <Text>This is the home screen of the app</Text>
      <Button
        onPress={() =>
          navigate('Profile', { names: ['Brent', 'Satya', 'Michaś'] })
        }
        title="Go to Brent's profile"
      />
    </View>
  );
}

goBack - Feche a tela ativa e volte

Optionally provide a key, which specifies the route to go back from. By default, goBack will close the route that it is called from: 

function ProfileScreen({ navigation: { goBack } }) {
  return (
    <View>
      <Button onPress={() => goBack()} title="Go back from ProfileScreen" />
    </View>
  );
}

Voltando de uma tela específica com o goBack

Considere o seguinte histórico da stack de navegação: 

navigation.navigate({ name: SCREEN, key: SCREEN_KEY_A });
navigation.navigate({ name: SCREEN, key: SCREEN_KEY_B });
navigation.navigate({ name: SCREEN, key: SCREEN_KEY_C });
navigation.navigate({ name: SCREEN, key: SCREEN_KEY_D });

Now you are on screen D and want to go back to screen A (popping D, C, and B). Then you can use navigate: 

navigation.navigate({ key: SCREEN_KEY_A }); // will go to screen A FROM screen D

Alternativamente, como a tela A é o topo da stack, você pode usar navigation.popToTop().

    <h2>
      Navigation events
    </h2>
    
    <p>
      Screens can add listeners on the <code>navigation</code> prop like in React Navigation. By default, there are 2 events available:
    </p>
    
    <ul>
      <li>
        <code>focus</code> - This event is emitted when the screen comes into focus
      </li>
      <li>
        <code>blur</code> - This event is emitted when the screen goes out of focus
      </li>
    </ul>
    
    <p>
      Exemplo:
    </p>
    
    <p>
      <samp id="simple-focus-and-blur"></p> 
      
      <pre><code class="js">function Profile({ navigation }) {

React.useEffect(() => { const unsubscribe = navigation.addListener('focus', () => { // do something });

return unsubscribe;

}, []);

return <ProfileContent />; } 

      <p>
        The <code>navigation.addListener</code> method returns a function to remove the listener which can be returned as the cleanup function in an effect.
      </p>
      
      <p>
        Each event listener receives an event object as it's argument. The event object contains following properties:
      </p>
      
      <ul>
        <li>
          <code>data</code> - Additional data regarding the event passed by the navigator. This can be <code>undefined</code> if no data was passed.
        </li>
        <li>
          <code>preventDefault</code> - Calling this method may prevent the default action performed by the action (such as switching tabs on <code>tabPress</code>). Support for preventing actions depends on the navigator and won't work for all events.
        </li>
      </ul>
      
      <p>
        Apart from <code>focus</code> and <code>blur</code>, each navigator can emit their own custom events. For example, stack navigator emits <code>transitionStart</code> and <code>transitionEnd</code> events, tab navigator emits <code>tabPress</code> event etc. You can find details about the events emitted on the individual navigator's documentation.
      </p>
      
      <h3>
        <code>isFocused</code> - Consultar o estado focalizado da tela
      </h3>
      
      <p>
        Retorna <code>true</code> se a tela estiver focada e <code>false</code> caso contrário.
      </p>
      
      <p>
        <samp id="use-is-focused"></p> 
        
        <pre><code class="js">const isFocused = navigation.isFocused();


        <p>
          This method doesn't re-render the screen when the value changes and mainly useful in callbacks. You probably want to use <a href="use-is-focused.md">useIsFocused</a> instead of using this directly, it will return a boolean a prop to indicating if the screen is focused.
        </p>
        
        <h2>
          Referência de API avançada
        </h2>
        
        <p>
          The <code>dispatch</code> function is much less commonly used, but a good escape hatch if you can't do what you need with the available methods such as <code>navigate</code>, <code>goBack</code> etc. We recommend to avoid using the <code>dispatch</code> method often unless absolutely necessary.
        </p>
        
        <h3>
          <code>dispatch</code> - Envie uma ação ao roteador
        </h3>
        
        <p>
          Use o despacho para enviar qualquer ação de navegação ao roteador. As outras funções de navegação usam o envio nos bastidores.
        </p>
        
        <p>
          Observe que, se você desejar disparar ações do react-navigation, deverá usar os criadores de ações fornecidos nesta biblioteca.
        </p>
        
        <p>
          See <a href="navigation-actions.md">Navigation Actions Docs</a> for a full list of available actions.
        </p>
        
        <pre><code class="js">import { CommonActions } from '@react-navigation/native';

navigation.dispatch( CommonActions.navigate({ name: 'Profile', params: {}, }) ); 

        <p>
          When dispatching action objects, you can also specify few additional properties:
        </p>
        
        <ul>
          <li>
            <code>source</code> - The key of the route which should be considered as the source of the action. The key maybe used by the router to handle the action. For example, the <code>replace</code> action will replace the route with the given key. By default, it'll use the key of the route that dispatched the action. You can explicitly pass <code>undefined</code> to override this behavior.
          </li>
          <li>
            <code>target</code> - The key of the navigation state the action should be applied on. By default, actions bubble to other navigators if not handled by a navigator. If <code>target</code> is specified, the action won't bubble if the navigator with the same key didn't handle it.
          </li>
        </ul>
        
        <p>
          Exemplo:
        </p>
        
        <pre><code class="js">import { CommonActions } from '@react-navigation/native';

navigation.dispatch({ ...CommonActions.navigate('Profile'), source: 'someRoutekey', target: 'someStatekey', }); 

        <h3>
          <code>dangerouslyGetParent</code> - obter navegador pai
        </h3>
        
        <p>
          Se, por exemplo, você tem um componente de tela que pode ser apresentado em vários navegadores, você pode usá-lo para influenciar seu comportamento com base em qual navegador ele está.
        </p>
        
        <p>
          Outro bom caso de uso para isso é encontrar o índice da rota ativa na lista de rotas dos pais. Portanto, no caso de uma stack, se você estiver no índice 0, talvez não queira renderizar um botão Voltar, mas se estiver em outro lugar na lista, você renderizará um botão Voltar.
        </p>
        
        <p>
          Certifique-se sempre de verificar se a chamada retorna um valor válido.
        </p>
        
        <h3>
          <code>dangerouslyGetState</code> - get state of navigator
        </h3>
        
        <p>
          Getting the navigator state could be useful in very rare situations. You most likely don't need to use this method. If you do, make sure you have a good reason.
        </p>
        
        <p>
          If you need the state for rendering content, you should use <a href="use-navigation-state.md"><code>useNavigationState</code></a> instead of this method.
        </p>
