createMaterialBottomTabNavigator

Uma barra de guias com tema de design de material na parte inferior da tela que permite alternar entre diferentes rotas. Rotas são inicializadas preguiçosamente - seus componentes de tela não são montados até que sejam focalizados primeiro.

To use this navigator, you need to install react-navigation-material-bottom-tabs:

npm install react-navigation-material-bottom-tabs react-native-paper

Essa API também requer que você instale react-native-vector-icons! Se você estiver usando a Expo, ela funcionará imediatamente. Caso contrário, siga estas instruções de instalação.

To use this tab navigator, import it from react-navigation-material-bottom-tabs

import { createMaterialBottomTabNavigator } from "react-navigation-material-bottom-tabs";

createMaterialBottomTabNavigator(
  RouteConfigs,
  MaterialBottomTabNavigatorConfig
);

Esta biblioteca usa o componente BottomNavigation do react-native-paper. If you configure the Babel plugin, it won't include the whole react-native-paper library in your bundle.

RouteConfigs

O objeto configs da rota é um mapeamento do nome da rota para uma configuração da rota.

MaterialBottomTabNavigatorConfig

  • shifting - Se o estilo de deslocamento é usado, a guia ativa parece mais ampla e as guias inativas não terão um rótulo. Por padrão, isso é true quando você tem mais de 3 guias.
  • labeled - Se deseja mostrar rótulos nas guias. Quando false, somente os ícones serão exibidos.
  • activeColor - Cor personalizada para ícone e rótulo na guia ativa.
  • inactiveColor - Cor personalizada para ícone e rótulo na guia inativa.
  • barStyle - Estilo para a barra de navegação inferior. Você pode definir um preenchimento inferior aqui se tiver uma barra de navegação translúcida no Android: barStyle = {{ paddingBottom: 48 }}.
  • initialRouteName - O nome da rota inicial a ser carregada.
  • order - Array de routeNames que define a ordem das guias.
  • paths - Forneça um mapeamento de routeName para path config, que substitui os caminhos definidos no routeConfigs.
  • backBehavior - initialRoute to return to initial tab, order to return to previous tab, history to return to last visited tab, or none. backBehavior - initialRoute para retornar à aba inicial, order para retornar à aba anterior, history para retornar à última aba visitada, ou none.

Exemplo:

export default createMaterialBottomTabNavigator({
  Album: { screen: Album },
  Library: { screen: Library },
  History: { screen: History },
  Cart: { screen: Cart },
}, {
  initialRouteName: 'Album',
  activeColor: '#f0edf6',
  inactiveColor: '#3e2465',
  barStyle: { backgroundColor: '#694fad' },
});

navigationOptions para telas dentro do navegador

title

Título genérico que pode ser usado como fallback para headerTitle e tabBarLabel.

tabBarIcon

React Element ou uma função que forneça { focused: boolean, horizontal: boolean, tintColor: string } retorna um React.Node, para exibir na barra de guias. horizontal é true quando o dispositivo está na paisagem e false quando retrato. O ícone é renderizado novamente sempre que a orientação do dispositivo é alterada.

tabBarColor

Cor para a barra de abas quando a aba correspondente à tela estiver ativa. Usado para o efeito de ondulação. Isso só é suportado quando shifting é true.

tabBarLabel

Cadeia de título de uma guia exibida na barra de guias. Quando indefinido, a cena title é usada. Para ocultar, veja a opção labeled na seção anterior.

tabBarAccessibilityLabel

Etiqueta de acessibilidade para o botão do separador. Isso é lido pelo leitor de tela quando o usuário toca na guia. É recomendável definir isso se você não tiver um rótulo para a guia.

tabBarTestID

ID para localizar este botão da guia nos testes.

tabBarOnPress

Callback para lidar com eventos de imprensa; o argumento é um objeto contendo:

  • navigation: navigation prop para a tela
  • defaultHandler: o manipulador padrão para tabulação

Useful for adding a custom logic before the transition to the next scene (the tapped one) starts. Ao definir tabBarOnPress, o defaultHandler precisa ser chamado para executar a ação padrão (ou seja, a switch tab).

Usando com react-native-paper (opcional)

Você pode usar o suporte a temas em react-native-paper para personalizar a navegação na parte inferior do material agrupando seu aplicativo em Provider de react-native-paper. Um caso de uso comum para isso pode ser personalizar a cor de fundo das telas quando o aplicativo tiver um tema escuro. Segue as instruções sobre a documentação do react-native-paper's para aprender a personalizar o tema.
