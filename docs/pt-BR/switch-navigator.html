createSwitchNavigator

O objetivo de SwitchNavigator é apenas mostrar uma tela por vez. Por padrão, ele não controla as ações de retorno e redefine as rotas para seu estado padrão quando você é desligado. Esse é o comportamento exato que queremos do fluxo de autenticação.

API Definition

createSwitchNavigator(RouteConfigs, SwitchNavigatorConfig);

RouteConfigs

O objeto configs da rota é um mapeamento do nome da rota para uma configuração da rota, que informa ao navegador o que apresentar para essa rota, consulte exemplo de createStackNavigator.

SwitchNavigatorConfig

Várias opções são passadas para o roteador subjacente para modificar a lógica de navegação:

  • initialRouteName - O nome da rota inicial a ser carregada.
  • navigationOptions - Navigation options for the navigator itself, to configure a parent navigator
  • defaultNavigationOptions - Default navigation options to use for screens
  • resetOnBlur - redefinir o estado de qualquer navegadores aninhados quando se muda para uma tela. O padrão é true.
  • paths - Forneça um mapeamento de routeName para path config, que substitui os caminhos definidos no routeConfigs.
  • backBehavior - initialRoute to return to initial route, order to return to previous route, history to return to last visited route, or none.
