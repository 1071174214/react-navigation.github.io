Moving between screens

Na seção anterior, "Olá React Navigation", definimos um "stack navigator" com duas rotas (Home e Detalhes), mas não aprendemos como permitir que um usuário navegue de Home para Details (embora tenhamos aprendido como alterar a inicial rota em nosso código, mas forçando nossos usuários a clonar nosso repositório e mudar a rota em nosso código, a fim de ver outra tela está, possivelmente, entre as piores experiências de usuário que se possa imaginar).

Se este fosse um navegador da Web, poderíamos escrever algo assim:

<a href="details.html">Vá a Detalhes</a>

Outra maneira de escrever isso seria:

<a onClick={() => { document.location.href = "details.html"; }}>Vá a Detalhes</a>

Faremos algo semelhante ao último, mas em vez de usar um documento document global, usaremos o prop navigation que é passado para os componentes da tela.

Navegando para uma nova tela

First navigation

import * as React from 'react';
import { Button, View, Text } from 'react-native';
import { createAppContainer } from 'react-navigation';
import { createStackNavigator } from 'react-navigation-stack';

class HomeScreen extends React.Component {
  render() {
    return (
      <View style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}>
        <Text>Home Screen</Text>
        <Button
          title="Go to Details"
          onPress={() => this.props.navigation.navigate('Details')}
        />
      </View>
    );
  }
}

// ... outro código da seção anterior

Vamos detalhar isso:

  • this.props.navigation: o prop navigation é passado para todos os componentes da tela (definição ) no "stack navigator" (mais sobre isso mais tarde em "Aprofundando no "navigation prop").
  • navigate('Details'): nós chamamos a função navigate(que está dentro do navigation prop — nomear é difícil!) com o nome da rota que gostaríamos de mover o usuário para.

Se chamarmos this.props.navigation.navigate com um nome de rota que não definimos em um "stack navigator", nada acontecerá. Dito de outra forma, só podemos navegar para rotas que foram definidas no nosso navegador de pilha — não podemos navegar para um componente arbitrário.

Então agora temos uma stack com duas rotas: 1) a rota Home 2) a rota Details. O que aconteceria se navegássemos novamente até a rota Details, na tela Details?

Navegue até uma rota multiplas vezes

class DetailsScreen extends React.Component {
  render() {
    return (
      <View style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}>
        <Text>Details Screen</Text>
        <Button
          title="Vá a detalhes... novamente"
          onPress={() => this.props.navigation.navigate('Details')}
        />
      </View>
    );
  }
}

Se você executar esse código, perceberá que quando tocar em "Vá a detalhes... mais uma vez "que não faz nada! Isso ocorre porque já estamos na rota Detalhes. A função navigate significa, grosso modo, "ir para esta tela", e se você já estiver nessa tela, faz sentido que não faça nada.

Vamos supor que realmente queremos adicionar outra tela de detalhes. Isso é muito comum nos casos em que você passa alguns dados exclusivos para cada rota (mais sobre isso depois, quando falamos sobre params!). Para fazer isso, podemos mudar navigate para push. Isso nos permite expressar a intenção de adicionar outra rota, independentemente do histórico de navegação existente.

push

<Button
  title="Go to Details... again"
  onPress={() => this.props.navigation.push('Details')}
/>

Cada vez que você chama push, adicionamos uma nova rota ao "stack navigator". Quando você chama navigate ele primeiro tenta encontrar uma rota existente com esse nome, e só envia uma nova rota se ainda não houver uma na "stack".

Voltando

O cabeçalho fornecido pelo "stack navigator" incluirá automaticamente um botão de voltar quando for possível voltar da tela ativa (se houver apenas uma tela no "stack navigator", não há nada para o qual você possa voltar e, portanto, não há botão "voltar).

Às vezes você desejará ser capaz de disparar programaticamente esse comportamento e, para isso, você pode usar this.props.navigation.goBack();.

goBack

class DetailsScreen extends React.Component {
  render() {
    return (
      <View style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}>
        <Text>Details Screen</Text>
        <Button
          title="Vá a detalhes... again"
          onPress={() => this.props.navigation.push('Details')}
        />
        <Button
          title="Go to Home"
          onPress={() => this.props.navigation.navigate('Home')}
        />
        <Button
          title="Go back"
          onPress={() => this.props.navigation.goBack()}
        />
      </View>
    );
  }
}

No Android, o React Navigation conecta-se ao botão de retorno do hardware e dispara a função goBack() para você quando o usuário a pressiona, para que ela se comporte como o usuário esperaria.

Outro requisito comum é poder voltar a múltiplas telas - por exemplo, se você tiver várias telas em uma "stack" e quiser dispensar todas elas para voltar à primeira tela. Neste caso, sabemos que queremos voltar para Home para que possamos usar navegar('Home') (não push! tente isso e veja a diferença). Outra alternativa seria navigation.popToTop(), que retorna para a primeira tela da "stack".

Sumário

  • this.props.navigation.navigate('RouteName') envia uma nova rota para o "stack navigator", se ainda não estiver na "stack", caso contrário, salta para a tela.
  • Nós podemos chamar this.props.navigation.push('RouteName') quantas vezes quisermos e continuaremos pressionando as rotas.
  • A barra de cabeçalho mostrará automaticamente um botão de retorno, mas você pode voltar programaticamente chamando this.props.navigation.goBack(). No Android, o botão de volta do hardware apenas funciona como esperado.
  • Você pode voltar para uma tela existente na "stack" com this.props.navigation.navigate ('RouteName'), e você pode voltar para a primeira tela na "stack" com this.props.navigation.popToTop().
  • A prop navigation está disponível para todos os componentes da tela (componentes definidos como telas na configuração da rota e renderizados pelo React Navigation como uma rota).
