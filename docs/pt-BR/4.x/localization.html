Localization

Inglês é apenas uma das muitas línguas que as pessoas falam ao redor do mundo (muito obrigado, Torre de Babel) e é educado e às vezes até mesmo necessário para traduzir para o seu aplicativo aos idiomas que seus usuários falam. Vejamos uma maneira de fazer isso no React Navigation - não é a única maneira, mas vai fazer o truque. Semelhante aos temas, usaremos screenProps. Você também pode usar a API de contexto do React, conforme demonstrado no guia Temas, para facilitar o acesso à função de conversão de vários componentes.

Configurando uma biblioteca de localização

Precisamos usar algum tipo de biblioteca para armazenar nossas traduções e fornecer uma função que nos dê acesso a elas, além de lidar com substitutos quando não tivermos um idioma específico definido. Localization and internationalization (i18n) are often used interchangeably, as in the example below where we get the current locale from expo-localization and use the i18n-js library for managing translations, for no particular reason other than it was available - use whatever you like.

import * as Localization from 'expo-localization'; // or whatever library you want
import i18n from 'i18n-js'; // or whatever library you want

const en = {
  foo: 'Foo',
  bar: 'Bar {{someValue}}',
};

const fr = {
  foo: 'Fou',
  bar: 'Bár {{someValue}}',
};

i18n.fallbacks = true;
i18n.translations = { fr, en };

// This will log 'en' for me, as I'm an English speaker
console.log(Localization.locale);

Conectando sua biblioteca de localização à navegação

Em seguida, vamos armazenar nosso código de idioma no estado de nosso componente de aplicativo raiz e, em seguida, encaminhá-lo por meio de screenProps para torná-lo disponível no React Navigation.

export default class App extends React.Component {
  state = {
    locale: Localization.locale,
  };

  setLocale = locale => {
    this.setState({ locale });
  };

  t = (scope, options) => {
    return i18n.t(scope, { locale: this.state.locale, ...options });
  };

  render() {
    return (
      <AppContainer
        screenProps={{
          t: this.t,
          locale: this.state.locale,
          setLocale: this.setLocale,
        }}
      />
    );
  }
}

Agora, em nossas telas, podemos usar estes screenProps da seguinte forma:

class Screen extends React.Component {
  static navigationOptions = ({ screenProps: { t } }) => ({
    title: t('foo'),
  });

  render() {
    let { t, locale } = this.props.screenProps;

    return (
      <View style={styles.container}>
        <Text style={styles.text}>
          Current locale: {locale}.{' '}
          {locale !== 'en' && locale !== 'fr'
            ? 'As traduções voltarão para "en" porque não há nenhuma disponível'
            : null}
        </Text>
        <Text>{t('bar', { someValue: Date.now() })}</Text>
        {locale === 'en' ? (
          <Button
            title="Trocar para Francês"
            onPress={() => this.props.screenProps.setLocale('fr')}
          />
        ) : (
          <Button
            title="Trocar para Inglés"
            onPress={() => this.props.screenProps.setLocale('en')}
          />
        )}
      </View>
    );
  }
}

Você pode executar este exemplo em este Snack. Novamente, você pode querer ir mais longe do que apenas passar isto através das screenProps se você quiser facilitar o acesso à função t ou a outra screenProps de qualquer componente React (e não apenas componentes de tela que são renderizados pelo React Navigation). Consulte os temas e a documentação do React sobre context para obter ajuda com isso.
