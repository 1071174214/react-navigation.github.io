Navigation prop reference

Cada componente screen no seu aplicativo é fornecido com o prop navigation automaticamente. O prop contém várias funções de conveniência que despacham ações de navegação no roteador da rota. Se parece com isso:

  • this.props.navigation
    • navigate - ir para outra tela, descobre a ação necessária para fazer isso
    • goBack - feche a tela ativa e mover para trás na stack
    • addListener - registrar atualizações para o ciclo de vida da navegação
    • isFocused - função que retorna true se a tela estiver focada e false caso contrário.
    • state - Atual state/rotas
    • setParams - fazer alterações nos parâmetros da rota
    • getParam - obter um parâmetro específico com fallback
    • dispatch - enviar uma ação ao roteador
    • dangerouslyGetParent - função que retorna o navegador pai, se houver

É importante destacar que a prop navigation não é passada para todos componentes; apenas componentes da tela recebem esta prop automaticamente! React Navigation não faz nada de mágico aqui. Por exemplo, se você fosse definir um componente MyBackButton e renderizá-lo como um filho de um componente de tela, você não seria capaz de acessar a prop navigation nele. Se, no entanto, você deseja acessar a prop navigation em qualquer um dos seus componentes, você pode usar o withNavigation HOC.

Funções dependentes do navegador

Existem várias funções adicionais presentes em this.props.navigation baseadas no tipo de navegador atual.

If the navigator is a stack navigator, several alternatives to navigate and goBack are provided and you can use whichever you prefer. The functions are:

  • this.props.navigation
    • push - empurrar uma nova rota para a stack
    • pop - volte na stack
    • popToTop - vai para o topo da stack
    • replace - substitua a rota atual por uma nova
    • reset - limpe o estado do navegador e substitua-o pelo resultado de várias ações
    • dismiss - descartar a stack atual

Se o navegador for um drawer navigator, os seguintes itens também estarão disponíveis:

  • this.props.navigation
    • openDrawer - abrir a drawer
    • closeDrawer - fechar a drawer
    • toggleDrawer - toggle the state, ie. switch from closed to open and vice versa

Referência de API comum

A grande maioria das suas interações com o prop da navigation envolverá navigate, goBack, state e setParams /getParam.

navigate - Link para outras telas

Chame isso para vincular a outra tela no seu aplicativo. Leva os seguintes argumentos:

navigation.navigate({ routeName, params, action, key })

Ou

navigation.navigate(routeName, params, action)

  • routeName - Um routeName de destino que foi registrado em algum lugar no roteador do aplicativo
  • params - Parâmetros para mesclar na rota de destino
  • action - (avançado) A sub-ação para executar no "router" filho, se a tela for um navegador. See Actions Doc for a full list of supported actions.
  • key - Optional identifier of what route to navigate to. Navigate back to this route, if it already exists
class HomeScreen extends React.Component {
  render() {
    const { navigate } = this.props.navigation;

    return (
      <View>
        <Text>This is the home screen of the app</Text>
        <Button
          onPress={() => navigate('Profile', { name: 'Brent' })}
          title="Vá para o perfil de Brent"
        />
      </View>
    );
  }
}

goBack - Feche a tela ativa e volte

Opcionalmente, forneça uma chave, que especifica a rota para a qual retornar. Por padrão, o goBack fechará a rota de onde é chamado. Se o objetivo é voltar em qualquer lugar, sem especificar o que está sendo fechado, chame .goBack(null); Observe que o parâmetro null é útil no caso de StackNavigators um dentro de outro para voltar em um navegador pai quando o navegador filho já tiver apenas um item na stack. Não se preocupe se isso for confuso, essa API precisa de algum trabalho.

Note -- a key is not the name of the route but the unique identifier you provided when navigating to the route. See navigation key.

class HomeScreen extends React.Component {
  render() {
    const { goBack } = this.props.navigation;
    return (
      <View>
        <Button onPress={() => goBack()} title="Go back from this HomeScreen" />
        <Button onPress={() => goBack(null)} title="Go back anywhere" />
        <Button
          onPress={() => goBack('key-123')}
          title="Go back from key-123"
        />
      </View>
    );
  }
}

Voltando de uma tela específica com o goBack

Considere o seguinte histórico da stack de navegação:

navigation.navigate({ routeName: SCREEN, key: SCREEN_KEY_A });
navigation.navigate({ routeName: SCREEN, key: SCREEN_KEY_B });
navigation.navigate({ routeName: SCREEN, key: SCREEN_KEY_C });
navigation.navigate({ routeName: SCREEN, key: SCREEN_KEY_D });

Now you are on screen D and want to go back to screen A (popping D, C, and B). Then you need to supply a key to goBack FROM:

navigation.goBack(SCREEN_KEY_B) // irá para a tela A da tela B

Alternativamente, como a tela A é o topo da stack, você pode usar navigation.popToTop().

addListener - Ouça as atualizações para o ciclo de vida da navegação

React Navigation emite eventos para os componentes da tela que os assinam:

  • willFocus - a tela receberá foco
  • didFocus - a tela focada (se houve uma transição, a transição foi concluída)
  • willBlur - a tela ficará desfocada
  • didBlur - a tela desfocada (se houve uma transição, a transição foi concluída)

Exemplo:

const didBlurSubscription = this.props.navigation.addListener(
  'didBlur',
  payload => {
    console.debug('didBlur', payload);
  }
);

// Remova o ouvinte quando terminar
didBlurSubscription.remove();

The JSON payload:

{
  action: { type: 'Navigation/COMPLETE_TRANSITION', key: 'StackRouterRoot' },
  context: 'id-1518521010538-2:Navigation/COMPLETE_TRANSITION_Root',
  lastState: undefined,
  state: undefined,
  type: 'didBlur',
};

Você também pode se inscrever em eventos de navegação declarativamente com o componente <NavigationEvents/>.

isFocused - Consultar o estado focalizado da tela

Retorna true se a tela estiver focada e false caso contrário.

let isFocused = this.props.navigation.isFocused();

Você provavelmente vai querer usar o withNavigationFocus ao invés de usar isso diretamente, ele passará um porp boolean isFocused para o seu componente.

state - Estado / rota atual da tela

A screen has access to its route via this.props.navigation.state. Each will return an object with the following:

{
  // o nome da configuração da rota no roteador
  routeName: 'profile',
  //um identificador exclusivo usado para classificar rotas
  key: 'main0',
  //um objeto opcional de opções de string para esta tela
  params: { hello: 'world' }
}

Isto é mais comumente usado para acessar o params para a tela, transmitido através de navigate ou setParams.

class ProfileScreen extends React.Component {
  render() {
    return <Text>Name: {this.props.navigation.state.params.name}</Text>;
  }
}

setParams - Faça alterações para rotear params

Disparar a ação setParams permite que uma tela altere os parâmetros na rota, o que é útil para atualizar os botões e título do cabeçalho. setParams funciona como o setState do React - ele mescla o objeto params fornecido com os parâmetros atuais.

class ProfileScreen extends React.Component {
  render() {
    return (
      <Button
        onPress={() => this.props.navigation.setParams({ name: 'Lucy' })}
        title="Definir nome do título para 'Lucy'"
      />
    );
  }
}

getParam - Obter um valor de parâmetro específico com um retorno

No passado, você pode ter encontrado o cenário assustador de acessar um param quando params é indefinido. Em vez de acessar o param diretamente, você pode chamar getParam.

Antes:

const { name } = this.props.navigation.state.params;

se params é undefined, isso falha

Depois:

const name = this.props.navigation.getParam('name', 'Peter');

if name or params are undefined, set the fallback to Peter.

Stack Actions

As ações a seguir funcionarão em qualquer Stack Navigator:

Push

Semelhante a navegar, o push moverá você para uma nova rota na stack. Isso difere de navigate em que navigate irá retornar ao início na "Stack" se uma rota do nome já estiver presente lá. O push sempre será adicionado por cima, então uma rota pode estar presente várias vezes.

navigation.push(routeName, params, action);
  • routeName - A destination routeName that has been registered somewhere in the app's router.
  • params - Params to merge into the destination route.
  • action - (avançado) A sub-ação para executar no "router" filho, se a tela for um navegador. See Actions Doc for a full list of supported actions.

Pop

Take you to the previous screen in the stack. If you provide a number, n, it will specify how many screens to take you back within the stack.

navigation.pop(n);

PopToTop

Chame isso para voltar para a rota principal na stack, dispensando todas as outras telas.

navigation.popToTop();

Replace

Chame isso para substituir a tela atual pela rota especificada, com params e sub-action.

navigation.replace(routeName, params, action);

Reset

Limpe o estado do navegador e substitua-o pelo resultado de várias ações.

navigation.reset([NavigationActions.navigate({ routeName: 'Profile' })], 0);

Dismiss

Chame isso se você estiver em uma stack dentro de outra(filho) e quiser descartar a stack inteira, retornando à stack pai.

navigation.dismiss();

Referência de API avançada

A função dispatch é muito menos usada, mas uma boa saída se você não puder fazer o que precisa com navigate e goBack.

dispatch - Envie uma ação ao roteador

Use dispatch to send any navigation action to the router. The other navigation functions use dispatch behind the scenes.

Observe que, se você desejar disparar ações do react-navigation, deverá usar os criadores de ações fornecidos nesta biblioteca.

Consulte Documentos de ações de navegação para obter uma lista completa de ações disponíveis.

import { NavigationActions } from 'react-navigation';

const navigateAction = NavigationActions.navigate({
  routeName: 'Profile',
  params: {},

  // navigate pode ter uma ação de navegação aninhada que será executada dentro do roteador filho
  action: NavigationActions.navigate({ routeName: 'SubProfileRoute' }),
});
this.props.navigation.dispatch(navigateAction);

dangerouslyGetParent - obter navegador pai

Se, por exemplo, você tem um componente de tela que pode ser apresentado em vários navegadores, você pode usá-lo para influenciar seu comportamento com base em qual navegador ele está.

Outro bom caso de uso para isso é encontrar o índice da rota ativa na lista de rotas dos pais. Portanto, no caso de uma stack, se você estiver no índice 0, talvez não queira renderizar um botão Voltar, mas se estiver em outro lugar na lista, você renderizará um botão Voltar.

Certifique-se sempre de verificar se a chamada retorna um valor válido.

class UserCreateScreen extends Component {
  static navigationOptions = ({ navigation }) => {
    const parent = navigation.dangerouslyGetParent();
    const gesturesEnabled =
      parent &&
      parent.state &&
      parent.state.routeName === 'StackWithEnabledGestures';

    return {
      title: 'New User',
      gesturesEnabled,
    };
  };
}
