Access the navigation prop from any component

withNavigation 是一个高阶组件，它可以将navigation这个 prop 传递到一个包装的组件。 当你无法直接将navigation 这个 prop 传递给组件，或者不想在深度嵌套的子组件中传递它时，它将非常有用。

默认情况下，不是屏幕组件的普通组件将不会接收 navigation props，例如在此MyBackButton组件中：

import React from 'react';
import { Button } from 'react-native';

export default class MyBackButton extends React.Component {
  render() {
    // This will throw an 'undefined is not a function' exception because the navigation
    // prop is undefined.
    return (
      <Button
        title="Back"
        onPress={() => {
          this.props.navigation.goBack();
        }}
      />
    );
  }
}

要解决这个问题, 你可以在页面中渲染它时传递 navigation prop 到 MyBackButton ,像这样: <MyBackButton navigation={this.props.navigation} />.

也可以使用 withNavigation 函数自动提供一个 navigation prop (如果你好奇,也可以使用 React Context). 该函数的行为与Redux的connect函数类似，虽然它不向派生的组件提供dispatch 这个 prop，但它提供了navigation 这个 prop。

import React from 'react';
import { Button } from 'react-native';
import { withNavigation } from 'react-navigation';

class MyBackButton extends React.Component {
  render() {
    return (
      <Button
        title="Back"
        onPress={() => {
          this.props.navigation.goBack();
        }}
      />
    );
  }
}

// withNavigation returns a component that wraps MyBackButton and passes in the
// navigation prop
export default withNavigation(MyBackButton);

使用这种方法，你可以在应用程序中的任何位置渲染MyBackButton，而无需明确传递navigation 这个 prop。
