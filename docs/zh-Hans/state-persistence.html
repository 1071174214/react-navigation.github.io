State persistence

你可能希望将用户的位置保存在应用程序中, 以便在应用程序重新启动后立即返回到同一位置。

这在开发过程中特别有用，因为它允许开发人员在刷新应用程序时保持在同一页面上。

注意：此功能目前被认为是实验性的，因为本页末尾列出了警告， 谨慎使用！

用法

您可以使用persistNavigationState和loadNavigationState prop 渲染顶级导航器的持久化：

  • persistNavigationState是一个异步函数，它接收单个参数-导航状态对象。 该功能应将其保留。
  • loadNavigationState是一个异步函数，它执行相反的操作-它应加载持久的导航状态并返回一个使用导航状态对象解析的Promise。 如果该函数拒绝，React Navigation将启动，就像没有提供任何状态一样。
const AppNavigator = createStackNavigator({...});
const persistenceKey = "persistenceKey"
const persistNavigationState = async (navState) => {
  try {
    await AsyncStorage.setItem(persistenceKey, JSON.stringify(navState))
  } catch(err) {
    // handle the error according to your needs
  }
}
const loadNavigationState = async () => {
  const jsonString = await AsyncStorage.getItem(persistenceKey)
  return JSON.parse(jsonString)
}

const App = () => <AppNavigator persistNavigationState={persistNavigationState} loadNavigationState={loadNavigationState} />;

开发模式

此功能在开发模式下特别有用。 你可以使用以下方法，有选择地启用它：

const AppNavigator = createStackNavigator({...});
function getPersistenceFunctions() {
  return __DEV__ ? {
    persistNavigationState: ...,
    loadNavigationState: ...,
  } : undefined;
}
const App = () => <AppNavigator {...getPersistenceFunctions()} />;

加载 View

由于状态是异步加载的，因此应用程序必须在loadNavigationState函数返回之前先渲染一个空白/加载视图。 要自定义在此时间内呈现的加载视图, 可以使用 renderLoadingExperimental prop:

<AppNavigator
  persistNavigationState={...}
  loadNavigationState={...}
  renderLoadingExperimental={() => <ActivityIndicator />}
/>

注意：此API可能在将来发生变化，这就是为什么它被标记为实验性的原因

警告：可序列化的 State

为了使此功能正常工作，每个参数，路线和导航状态都必须完全可序列化。 通常，会将状态序列化为JSON字符串。 这意味着你的 route 和 param 必须不包含函数、类实例或递归数据结构。

如果需要修改导航状态对象，则可以在loadNavigationState/persistNavigationState函数中进行，但是请注意，如果您的loadNavigationState提供了无效的对象（无法从其恢复导航状态的对象），React Navigation可能无法优雅地处理这种情况。

警告：Route/Router 定义更改

当你的应用程序代码更改为在导航状态下支持给定路线的新路线或不同路线时，应用程序可能会在呈现旧导航状态时中断。

This may happen regularly during development as you re-configure your routes and navigator hierarchy. 但是当你发布一个新版本的应用程序时，它也可能在生产中发生！

保守的行为是在应用程序更新后擦除导航状态。 最简单的方法是为你发布给用户的每个版本引用不同的持久性密钥。

React Navigation 使用 React 的componentDidCatch功能试图减轻由路由定义更改引起的崩溃，但这被认为是实验性的，可能无法捕获所有错误。
