Supporting safe areas

默认情况下，React Navigation有助于确保你的应用程序在iPhone X和其他带有缺口和“安全区域”的设备上正确显示。 它通过在可能与传感器集群（“齐刘海”）或 Home 主页指示器交互的UI元素内部使用SafeAreaView来实现。

目标是（a）最大化屏幕的使用率（b）而不隐藏内容，也不会因物理显示切口或某些操作系统UI遮住内容而使其难以与之交互。

试图通过将整个应用程序包装在带有填充物的容器中来解决问题（a），以确保所有内容都不会被遮挡。 但是这样做会浪费屏幕上的一堆空间，如左下方的图片所示。 我们理想中想要的是右侧图片。 我们可以为此使用SafeAreaView。 本指南的其余部分提供了有关如何在React Navigation中支持安全区域的更多信息。

隐藏/自定义导航栏或选项卡栏

但是，如果你要覆盖默认导航栏，请务必确保你的用户界面不会对这些硬件元素中的任何一个造成干扰。

例如，如果我不为header或tabBarComponent提供任何内容，则不会渲染任何内容

const Tabs = createBottomTabNavigator({
  ...
}, {
  tabBarComponent: () => null,
});

export default createStackNavigator({
  ...
}, {
  headerMode: 'none',
});

要解决这个问题，你可以将你的内容包装在一个SafeAreaView中，它可以从react-navigation导入。 请注意，SafeAreaView 不能包装整个导航器, 只能是页面组件或别的内容。

import SafeAreaView from 'react-native-safe-area-view';

class MyHomeScreen extends Component {
  render() {
    return (
      <SafeAreaView style={styles.container}>
        <Text style={styles.paragraph}>This is top text.</Text>
        <Text style={styles.paragraph}>This is bottom text.</Text>
      </SafeAreaView>
    );
  }
}

这将检测应用程序是否在 iPhone X 上运行，如果是，确保内容不会被任何硬件元素遮挡。

Landscape Mode（横屏模式）

即使你的应用程序在横屏模式下使用默认的 navigation bar 和 tab bar，确保内容不会被齐刘海遮挡也很重要。

为了修复这个问题，你可以再次使用SafeAreaView包裹你的内容， 在竖屏模式下，这不会和 `navigation bar</ code> 以及 tab bar</ code> 的默认行为产生冲突。

总之，请在使用 React Navigation 导航器注册的页面上使用 SafeAreaView `组件。

使用 forceInset 获得更多控制

在某些情况下，您可能需要通过 padding 进行更多地控制， 例如，你可以通过向 SafeAreaView 传递 forceInset 来删除 底部的 padding。

<SafeAreaView style={styles.container} forceInset={{ bottom: 'never' }}>
  <Text style={styles.paragraph}>This is top text.</Text>
  <Text style={styles.paragraph}>This is bottom text.</Text>
</SafeAreaView>

forceInset使用 key（取值：top | bottom | left | right | vertical | horizontal）和value（取值：'always' | 'never'）组成一个对象 或者，您可以通过传递一个整数来完全重写填充。

Android 缺口

React Native当前不公开API来访问有关Android设备上设备保险开关的信息。 如果您的应用程序具有不透明的状态栏（React Native中的默认状态栏），则可以处理设备具有切口的区域，而无需任何进一步的工作。 如果不是，要解决此问题，你可能需要使用以下临时解决方案：

  • 安装 react-native-device-info.
  • 使用DeviceInfo.hasNotch()检查设备是否具有槽口-这会将设备品牌和型号与带有槽口的设备列表进行比较-粗略但有效的解决方法。
  • 如果设备有槽口，则可能需要通过执行以下操作来增加 SafeAreaView 已知的状态栏高度：
import { Platform } from 'react-native';
import SafeAreaView from 'react-native-safe-area-view';
import DeviceInfo from 'react-native-device-info';

if (Platform.OS === 'android' && DeviceInfo.hasNotch()) {
  SafeAreaView
    .setStatusBarHeight
    /* Some value for status bar height + notch height */
    ();
}
