Deep linking

在本指南中，我们将设置我们的应用程序来处理外部 URI。 假设我们需要使用，像mychat://chat/Eric这样的URI来打开我们的应用程序，并直接链接到名为“Eric”的某个用户的聊天页面。

配置

以前，我们已经定义了一个这样的导航器：

const SimpleApp = createAppContainer(
  createStackNavigator({
    Home: { screen: HomeScreen },
    Chat: { screen: ChatScreen },
  })
);

我们希望使用，像chat/Eric这样的路径，链接到 “Chat” 页面，并将user作为参数传递过去。 让我们用path </ code>重新配置我们的 “Chat” 屏幕，这个<code> path可以告诉路由应该匹配什么相对路径，以及应该提取哪些参数。 这个路径规范是 chat/:user。

const SimpleApp = createAppContainer(
  createStackNavigator({
    Home: { screen: HomeScreen },
    Chat: {
      screen: ChatScreen,
      path: 'chat/:user',
    },
  })
);

如果我们有嵌套的导航器，则需要为每个父页面提供一个path。 所有路径都将被串联，也可以是一个空字符串。 这个路径规范是 chat/:user。

const AuthSwitch = createAppContainer(
  createStackNavigator({
    AuthLoading: { screen: AuthLoadingScreen },
    App: {
      screen: AppStack,
      path: '',
    },
    Auth: { screen: AuthStack },
  })
);

const AppStack = createStackNavigator({
  Home: { screen: HomeScreen },
  Friends: {
    screen: FriendsScreen,
    path: 'friends',
  },
});

const FriendsScreen = createStackNavigator({
  Overview: { screen: OverviewScreen },
  Chat: {
    screen: ChatScreen,
    path: 'chat/:user',
  },
});

创建 Expo 项目

First, you will want to specify a URL scheme for your app. This corresponds to the string before :// in a URL, so if your scheme is mychat then a link to your app would be mychat://. 该方案仅适用于独立应用, 您需要重新构建独立应用, 更改才能生效。 在 Expo 客户端应用中, 你可以使用 exp://ADDRESS:PORT 进行深度链接, 其中 ADDRESS 通常是 127.0.0.1, PORT 通常是 19000 - 当运行 expo start 时, URL 会被打印出来. 如果你想测试自定义的方案,需要运行 expo build:ios -t simulator 或 expo build:android ,将编译后的二进制文件安装到你的模拟器上 你可以通过 scheme key, 在 app.json中注册一个方案

{
  "expo": {
    "scheme": "mychat"
  }
}

URI前缀

接下来，让我们配置我们的导航容器，使其能够从传入应用程序的 URI 中提取路径。

import { Linking } from 'expo';

const SimpleApp = createAppContainer(createStackNavigator({...}));

const prefix = Linking.makeUrl('/');

const MainApp = () => <SimpleApp uriPrefix={prefix} />;

需要使用 Expo.Linking.makeUrl 的原因是, 根据您是在客户端应用中还是在独立应用中, 方案会有所不同。

在 iOS 上测试深度链接

要在 Expo 客户端应用中测试模拟器上的 URI, 请运行以下操作:

xcrun simctl openurl booted [ put your URI prefix in here ]

// for example

xcrun simctl openurl booted exp://127.0.0.1:19000/--/chat/Eric

在 Android 上测试深度链接

To test the intent handling in the Expo client app on Android, run the following:

adb shell am start -W -a android.intent.action.VIEW -d "[ put your URI prefix in here ]" host.exp.exponent

# for example

adb shell am start -W -a android.intent.action.VIEW -d "exp://127.0.0.1:19000/--/chat/jane" host.exp.exponent

Change host.exp.exponent to your app package name if you are testig on a standalone app.

阅读expo 链接指南，了解更多关于如何配置 expo 项目链接的信息。

使用 react-native init 创建项目

URI前缀

接下来，让我们配置我们的导航容器，使其能够从传入应用程序的 URI 中提取路径。

const SimpleApp = createAppContainer(createStackNavigator({...}));

const prefix = 'mychat://';

const MainApp = () => <SimpleApp uriPrefix={prefix} />;

iOS

让我们将原生的 iOS 应用程序配置为使用 mychat:// URI 方案打开。

In SimpleApp/ios/SimpleApp/AppDelegate.m:

// Add the header at the top of the file:
#import <React/RCTLinkingManager.h>

// Add this above the `@end`:

- (BOOL)application:(UIApplication *)app openURL:(NSURL *)url
            options:(NSDictionary<UIApplicationOpenURLOptionsKey,id> *)options
{
  return [RCTLinkingManager application:app openURL:url options:options];
}

在Xcode中，在SimpleApp/ios/SimpleApp.xcodeproj处打开项目。 选择侧边栏中的项目并导航到信息选项卡。 滚动到 "URL Types" 并添加一项。 In the new URL type, set the identifier and the URL scheme to your desired URL scheme.

现在你可以在Xcode中运行项目，或者在命令行上 re-build：

react-native run-ios

要在模拟器上测试 URI，请运行以下命令：

xcrun simctl openurl booted mychat://chat/Eric

要在真机上测试 URI，请打开 Safari 并输入mychat://chat/Eric。

Android

要在 Android 中配置外部链接，您可以在清单文件中创建新的 intent。

In SimpleApp/android/app/src/main/AndroidManifest.xml, do these followings adjustments:

  1. 为了在已经存在的 MainActivity 上接收 intent, 请设置 MainActivity 的 launchMode 为 singleTask 如果你想在已经注册的 - details 页面中使用深度链接进行导航, 这么做是有用的
  2. 在 MainActivity 入口处添加新的 intent-filter 类型为 VIEW:
<activity
    android:name=".MainActivity"
    android:launchMode="singleTask">
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
    </intent-filter>
    <intent-filter>
        <action android:name="android.intent.action.VIEW" />
        <category android:name="android.intent.category.DEFAULT" />
        <category android:name="android.intent.category.BROWSABLE" />
        <data android:scheme="mychat" />
    </intent-filter>
</activity>

现在，重新安装应用

react-native run-android

要测试Android中的 intent 处理，请运行以下命令：

adb shell am start -W -a android.intent.action.VIEW -d "mychat://chat/Eric" com.simpleapp

禁用 deep linking

In case you want to handle routing with deep-linking by yourself instead of react-navigation, you can pass enableURLHandling={false} prop to your app container:

const SimpleApp = createAppContainer(createStackNavigator({...}));

const MainApp = () => <SimpleApp enableURLHandling={false} />;
