自定义 Android Back 键的行为

默认情况下，当用户按下 Android 物理返回键时，react-navigation 会返回到新页面，如果没有可返回的页面，则退出应用。 这是一个合理的默认行为，但有些情况下你可能想要实现自定义处理。

例如，在一个页面中，用户正在列表中选择某一项，并且“选择模式”处于活跃状态。 在按下返回按钮时，你首先需要停用“选择模式”，然后在第二次按下返回键时，返回上一页。 以下代码片段演示了这种情况。 We make use of BackHandler which comes with react-native along with the useFocusEffect hook to add our custom hardwareBackPress listener.

Returning true from onBackPress denotes that we have handled the event, and react-navigation's listener will not get called, thus not popping the screen. 返回false会该方法继续执行 - react-navigation 的事件监听器将销毁当前页面。

function ScreenWithCustomBackBehavior() {
  // ...

  const backHandler = React.useCallback(() => {
    const onBackPress = () => {
      if (isSelectionModeEnabled()) {
        disableSelectionMode();
        return true;
      } else {
        return false;
      }
    };

    BackHandler.addEventListener('hardwareBackPress', onBackPress);

    return () =>
      BackHandler.removeEventListener('hardwareBackPress', onBackPress);
  }, [isSelectionModeEnabled, disableSelectionMode]);

  useFocusEffect(backHandler);

  // ...
}

所提出的方法对于显示在StackNavigator中的页面很适用。 在某些情况下，自定义返回键处理可能暂时不被支持（例如， 目前已知的一种情况就是：在打开的 drawer 中进行返回按钮的事件处理。 如若遇到类似情况，欢迎给我们提 PR）。

为什么不使用组件的生命周期方法？

At first, you may be inclined to use componentDidMount to subscribe for the back press event and componentWillUnmount to unsubscribe, or use useEffect to add the listener. 这种方法不会生效的-想了解更多请参阅navigation lifecycle。
