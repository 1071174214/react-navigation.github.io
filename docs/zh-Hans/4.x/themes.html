Themes

提供浅色主题和深色主题是让用户根据一天中的时间或他们的喜好调整应用外观的好方法。 它还表明你是一位时髦的应用程序开发人员，紧跟当今的趋势。

内置的主题

注意: 使用内置的主题需要 react-navigation@>=3.12.0!

随着操作系统增加了对亮和暗模式的内置支持，支持暗模式不再是跟上潮流，而是更多地符合用户对应用程序工作方式的平均期望。 为了与操作系统默认值合理一致的方式提供对亮和暗模式的支持，这些主题内置于React Navigation中。 您可以将theme prop 传递给你的 App Container 组件，以便在明暗模式之间切换，并且该theme prop 的值可以来自你使用的任何 API 从应用程序用户界面内的自定义配置中检测暗模式（如果是较旧的操作系统版本）的用户首选项。

let Navigation = createAppContainer(RootStack);

// `theme` can be `light` or `dark`. It defaults to `light` if not specified.
export default () => <Navigation theme="light" />;

这将为你的 stack navigator, bottom tab navigator, 和 drawer navigator 添加样式 React Navigation还提供了多种工具来帮助你自定义这些导航器，并且导航器中的页面也支持这两个主题。

使用操作系统首选项

在撰写本文时，react-native当前不支持检测操作系统的配色方案首选项（你可以 Follow 这个 PR）。 在它成为核心库的一部分并更新到包含它的版本之前，你可以先使用react-native-appearance。

需要iOS 13才能真正通过系统设置切换黑暗模式。

注意：如果你使用 Expo 管理的工作流程，则需要使用SDK 35+

首先,你需要安装 react-native-appearance. 请按照README中的说明进行操作.

安装后，请按照以下步骤设置根组件：

import { AppearanceProvider, useColorScheme } from 'react-native-appearance';

// Other navigation code goes here...
let Navigation = createAppContainer(RootStack);

export default () => {
  let theme = useColorScheme();

  return (
    <AppearanceProvider>
      <Navigation theme={theme} />
    </AppearanceProvider>
  )
}

如果你使用的 React Native 版本尚不支持hooks, 你可以使用Appearance.addChangeListener(cb) 和 Appearance.getColorScheme() 函数, 像README 中描述的那样.

在 react-navigation/theme-example 中查看主题集成的完成示例.

使用当前选择的主题

有两种工具可用于访问来自应用程序导航容器的任何组件中的主题：useTheme和ThemeConext。

useTheme是返回主题的简单自定义钩子。

import * as React from 'react';
import { TouchableOpacity, Text } from 'react-native';
import { useTheme } from 'react-navigation';

// Black background and white text in light theme, inverted on dark theme
function MyButton() {
  let theme = useTheme();

  return (
    <TouchableOpacity
      style={{ backgroundColor: theme === 'light' ? '#000' : '#fff' }}
    >
      <Text style={{ color: theme === 'light' ? '#fff' : '#000' }}>
        Button!
      </Text>
    </TouchableOpacity>
  );
}

ThemeContext允许你使用ThemeContext.Consumer模式或static contextType访问主题。

import * as React from 'react';
import { TouchableOpacity, Text } from 'react-native';
import { ThemeContext } from 'react-navigation';

function MyButton() {
  return (
    <ThemeContext.Consumer>
      {theme => (
        <TouchableOpacity
          style={{ backgroundColor: theme === 'light' ? '#000' : '#fff' }}>
          <Text style={{ color: theme === 'light' ? '#fff' : '#000' }}>Button!</Text>
        </TouchableOpacity>
      )}
    <ThemeContext.Consumer>
  );
}

import * as React from 'react';
import { TouchableOpacity, Text } from 'react-native';
import { ThemeContext } from 'react-navigation';

class MyButton extends React.Component {
  static contextType = ThemeContext;

  render() {
    const theme = this.context;
    return (
      <TouchableOpacity
        style={{ backgroundColor: theme === 'light' ? '#000' : '#fff' }}
      >
        <Text style={{ color: theme === 'light' ? '#fff' : '#000' }}>
          Button!
        </Text>
      </TouchableOpacity>
    );
  }
}

使用默认主题色

有一个很小但可能有用的颜色列表，用于根据主题设置导航器样式。 此颜色列表在ThemeColors下导出。 有关颜色的完整列表，请参见 TypeScript 定义。

import * as React from 'react';
import { TouchableOpacity, Text } from 'react-native';
import { ThemeColors, useTheme } from 'react-navigation';

function MyButton() {
  let theme = useTheme();
  let colors = ThemeColors[theme];

  return (
    <TouchableOpacity style={{ backgroundColor: colors.bodyContent }}>
      <Text style={{ color: colors.label }}>Button!</Text>
    </TouchableOpacity>
  );
}

默认主题组件

几个组件的默认设置都偏向特定主题。 例如: Text, 默认是黑暗主题. StatusBar 默认是黑色文本 React Navigation提供了主题替代方案。

import * as React from 'react';
import { TouchableOpacity, Text } from 'react-native';
import { Themed } from 'react-navigation';

function MyButton() {
  return (
    <View style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}>
      <TouchableOpacity style={{ backgroundColor: colors.bodyContent }}>
        <Themed.Text>Button!</Themed.Text>
      </TouchableOpacity>
      <Themed.StatusBar />
    </View>
  );
}

静态导航器配置中的内置主题

现在可以使用light和dark 关键字将导航器的静态配置选项中指定的颜色指定为对象：

let Tabs = createBottomTabNavigator(
  {
    /* routes */
  },
  {
    tabBarOptions: {
      activeTintColor: {
        light: '#000',
        dark: '#fff',
      },
      inactiveTintColor: {
        light: 'rgba(0,0,0,0.2)',
        dark: 'rgba(255,255,255,0.2)',
      },
    },
  }
);

旧的格式仍然可以使用，但是以以下方式指定的颜色将无法适应主题：

let Tabs = createBottomTabNavigator(
  {
    /* routes */
  },
  {
    tabBarOptions: {
      activeTintColor: '#000',
      inactiveTintColor: 'rgba(0,0,0,0.2)',
    },
  }
);

使用 React Context 的自定义主题

你可能需要比内置主题更多的控制权。 在这种情况下，你可以完全从头构建自己的主题。

使用 React Navigation 将自定义主题构建到应用程序中与没有应用程序的 React 应用程序没有太大不同；主要区别在于你需要使用screenProps来更新由navigationOptions控制的样式属性，并且当在导航器配置中控制样式属性时，我们需要另一种方法。 首先，我们将概述不使用 React Navigation 的 React 应用的主题，然后我们将更深入地探讨这些差异。 此外，本指南假设你已经对 React Navigation 有所了解，尤其是如何使用和配置导航器。

React's context API allows you to share state from an ancestor component to any of its descendants without explicitly passing the value through layers and layers of components ("prop drilling"). 这是构建主题的有用工具，因为我们可以在应用程序的根目录定义主题，然后从其他任何地方访问主题，并在主题更改时重新渲染每个主题组件。 如果你不熟悉如何使用上下文，可能需要先阅读反应文档，然后再继续。

import * as React from 'react';
import { Text, TouchableOpacity, View } from 'react-native';

const ThemeContext = React.createContext(null);
const ThemeConstants = {
  light: {
    backgroundColor: '#fff',
    fontColor: '#000',
  },
  dark: {
    backgroundColor: '#000',
    fontColor: '#fff',
  },
};

export default class AppContainer extends React.Component {
  state = {
    theme: 'light',
  };

  toggleTheme = () => {
    this.setState(({ theme }) => ({
      theme: theme === 'light' ? 'dark' : 'light',
    }));
  };

  render() {
    return (
      <ThemeContext.Provider
        value={{ theme: this.state.theme, toggleTheme: this.toggleTheme }}
      >
        <HomeScreen />
      </ThemeContext.Provider>
    );
  }
}

class HomeScreen extends React.Component {
  render() {
    return (
      <ThemedView
        style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}
      >
        <ThemeContext.Consumer>
          {({ toggleTheme }) => (
            <ThemedButton title="Toggle theme" onPress={toggleTheme} />
          )}
        </ThemeContext.Consumer>
      </ThemedView>
    );
  }
}

class ThemedButton extends React.Component {
  render() {
    let { title, ...props } = this.props;
    return (
      <TouchableOpacity {...props}>
        <ThemeContext.Consumer>
          {({ theme }) => (
            <Text style={{ color: ThemeConstants[theme].fontColor }}>
              {title}
            </Text>
          )}
        </ThemeContext.Consumer>
      </TouchableOpacity>
    );
  }
}

class ThemedView extends React.Component {
  render() {
    return (
      <ThemeContext.Consumer>
        {({ theme }) => (
          <View
            {...this.props}
            style={[
              this.props.style,
              { backgroundColor: ThemeConstants[theme].backgroundColor },
            ]}
          />
        )}
      </ThemeContext.Consumer>
    );
  }
}

这一堆代码, 除了在上下文中传递主题然后在主题组件中需要主题时将其从上下文中拉出之外，这里没有什么其他事情可做。 诸如ThemedView和ThemedButton之类的主题组件可帮助你避免不断重复主题上下文相关的样板。

navigationOptions中的主题

当前实现navigationOptions的一个令人遗憾的局限是，我们无法访问 React Context 以用于headerStyle和headerTintColor等属性。 我们可以并且应该在访问 React 组件的属性中使用它们，例如在headerRight中，我们可以提供类似ThemedHeaderButton的组件。 要将主题应用于其他属性，我们需要使用screenProps。

import { 
  createAppContainer,
  createStackNavigator,
  ThemeContext
} from 'react-navigation';

class HomeScreen extends React.Component {
  static navigationOptions = ({ screenProps }) => {
    let currentTheme = ThemeConstants[screenProps.theme];

    return {
      title: 'Home',
      headerTintColor: currentTheme.fontColor,
      headerStyle: { backgroundColor: currentTheme.backgroundColor },
    };
  };

  render() {
    return (
      <ThemedView
        style={{ flex: 1, alignItems: 'center', justifyContent: 'center' }}
      >
        <ThemeContext.Consumer>
          {({ toggleTheme }) => (
            <ThemedButton title="Toggle theme" onPress={toggleTheme} />
          )}
        </ThemeContext.Consumer>
      </ThemedView>
    );
  }
}

const Stack = createStackNavigator({ Home: HomeScreen });
const Navigation = createAppContainer(Stack);

export default class AppContainer extends React.Component {
  state = {
    theme: 'light',
  };

  toggleTheme = () => {
    this.setState(({ theme }) => ({
      theme: theme === 'light' ? 'dark' : 'light',
    }));
  };

  render() {
    return (
      <ThemeContext.Provider
        value={{ theme: this.state.theme, toggleTheme: this.toggleTheme }}
      >
        <Navigation screenProps={{ theme: this.state.theme }} />
      </ThemeContext.Provider>
    );
  }
}

👏 现在，当主题更改时，堆栈标题样式将更新。

注意：将来我们将弃用screenProps并将其完全移至 React Context。 目前，screenProps是执行此操作的最佳方法，并且当此更改发生时，将很容易迁移。

主题标签和其他类似的导航器

某些导航器在初始化时可能会在导航器配置对象中配置了其样式。 虽然最好更新这些导航器，以便可以通过navigationOptions对其进行更轻松地配置，但前提是它们允许我们覆盖使用自己的组件呈现的UI并允许我们访问默认组件，我们可以和他们一起工作。 我们将研究如何为底部标签导航器设置主题。

import {
  createAppContainer,
  createStackNavigator,
  createBottomTabNavigator,
  BottomTabBar,
  ThemeContext
} from 'react-navigation';

const ThemeConstants = {
  light: {
    backgroundColor: '#fff',
    fontColor: '#000',
    activeTintColor: 'blue',
    inactiveTintColor: '#ccc',
  },
  dark: {
    backgroundColor: '#000',
    fontColor: '#fff',
    activeTintColor: '#fff',
    inactiveTintColor: '#888',
  },
};

// Notice how we override the `activeTintColor`, `inactiveTintColor` and
// `backgroundColor` of the tab bar with our theme styles.
class ThemedBottomTabBar extends React.Component {
  render() {
    return (
      <ThemeContext.Consumer>
        {({ theme }) => (
          <BottomTabBar
            {...this.props}
            activeTintColor={ThemeConstants[theme].activeTintColor}
            inactiveTintColor={ThemeConstants[theme].inactiveTintColor}
            style={{
              backgroundColor: ThemeConstants[theme].backgroundColor,
            }}
          />
        )}
      </ThemeContext.Consumer>
    );
  }
}

const Stack = createStackNavigator({ Home: HomeScreen });
const Tabs = createBottomTabNavigator(
  { Stack },
  { tabBarComponent: ThemedBottomTabBar }
);
const Navigation = createAppContainer(Tabs);
