Integrating with MobX State Tree

本指南探讨了在使用 MobX State Tree（MST）进行状态管理在使用 React Navigation 的 React Native 项目中的可能方法。。 该指南随附了示例应用。 该指南的某些部分可能也与 MobX 的用户有关，但是请注意，MobX并未随附内置的解决（反）序列化其状态的方法。

请注意，在本指南中，Mobx状态树不用于管理导航状态本身-仅用于导航参数！

概述

本指南的目标是将MST与React Navigation结合使用，并获得最佳的开发人员体验。 在本指南的范围内，这意味着允许我们执行完整的JS重装，并使其回到重装发生之前的状态。

为此，我们将使用React Navigation的内置机制保留导航状态。 我们还需要保留应用程序状态和导航参数-这样，当您在应用程序中的屏幕上工作并进行完整的JS重新加载时，您将被带回到具有相同数据的同一页面。

指北

首先，从创建初始导航结构和React组件开始。 完成后，继续在MST中对状态建模。 如果您想了解更多有关此的信息，请查看 egghead.io课程。

在这一点上，你可能想知道如何将MST对象与组件连接。 答案在 mobx-react 中，该包包含用于MobX的React绑定（它们也适用于MST）。 你可能会使用Provider组件以及inject和observer函数。

使用Provider封装从根组件的render方法返回的内容：

<Provider myObject={this.myObject}>
  <AppNavigator />
</Provider>

这将允许通过inject函数从应用程序中的任何React组件访问myObject。

使用observer函数包装呈现可观察数据的所有组件。 这将确保一旦组件渲染的数据发生更改，组件就会重新渲染。

Navigation params

应用程序中的屏幕通常取决于参数。 通过React Navigation，您可以将参数从一个页面发送到另一个页面。 这些参数以导航状态存储。 但是，为了保持导航状态，它需要可序列化。 MST不能很好地满足此要求，因为MST对象是复杂的对象，而且React Navigation不知道如何对其进行反序列化。 在本指南中，我们将通过自己存储导航参数来解决此问题。

这意味着，而不是将参数从一个页面发送到另一个页面（例如， 使用props.navigation.navigate（'MyScreen'，{complexMSTObject}）），我们会将参数存储到导航存储中，然后在不发送任何参数的情况下进行导航，然后在目标页面上选择导航商店中的参数。

举个例子，导航存储看起来可能类似于：

import { types, onSnapshot, getRoot } from 'mobx-state-tree';
import { Product } from '../models/Product';
import { User } from '../models/User';

export const NavigationStore = types
  .model('NavigationStore', {
    productDetailScreenParams: types.map(
      types.model('ProductDetailScreenParams', {
        product: types.optional(types.safeReference(Product)),
      })
    ),
    userProfileScreenParams: types.model('UserProfileScreenParams', {
      user: types.maybe(types.safeReference(User)),
    }),
  })
  .actions(self => ({
    ...
  }));

请注意，userProfileScreenParams是带有user条目的简单模型，而productDetailScreenParams是ProductDetailScreenParams模型的映射。 我们选择这种数据形状的原因是，我们的应用程序中只有一个用户配置文件页面，该屏幕从userProfileScreenParams读取其参数。 productDetailScreenParams是一张地图，因为该应用程序可以在堆栈上具有多个产品页面。 每个页面都指向地图中保存的Product实例。 地图中的 key 是React Navigation key：将key视为路线的标识符。

您的导航存储区也可能只是一张地图，其中每个页面（无论它是产品屏幕还是用户个人资料页面）都将存储其导航参数。 这是示例应用中采用的方法。

摘要

  • 可以在React Native应用程序中将React Navigation与MobX状态树一起使用
  • 使用Provider组件以及inject和observer函数使用React连接MobX或MST
  • 可以保留整个应用程序状态并在JS重新加载后将其还原
