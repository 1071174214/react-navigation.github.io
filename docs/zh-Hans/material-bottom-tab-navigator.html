createMaterialBottomTabNavigator

屏幕底部的材料设计主题标签栏，可让您在不同路由之间切换。 路由被懒加载 - 它们的屏幕组件直到第一次获取焦点时才被加载。

要使用此导航器，请确保已安装 react-navigation及其依赖项，然后安装<a href =“https://github.com/react- navigation/material-bottom-tabs”> react-navigation-material-bottom-tabs 和react-native-paper。

yarn add react-navigation-material-bottom-tabs react-native-paper

此API还要求您安装react-native-vector-icons！ 如果您使用的是Expo（托管或裸机），请改为运行expo install@expo/vector-icons。 否则，请按照这些安装说明来操作。

API

import { createMaterialBottomTabNavigator } from 'react-navigation-material-bottom-tabs';

createMaterialBottomTabNavigator(
  RouteConfigs,
  MaterialBottomTabNavigatorConfig
);

使用 react-native-paper 的 BottomNavigation 组件的一个库， 如果你配置Babel插件，它将不包含整个react-native-paper库。

RouteConfigs

路由配置对象是从路由名称到路由配置的映射。

MaterialBottomTabNavigatorConfig

  • shifting-无论是否使用shifting样式，活动选项卡都会显示得更宽，而非活动选项卡将没有标签。 默认情况下，当你有3个以上的标签时，该标签为true。
  • labeled-是否在标签中显示标签。 如果false，则仅显示图标。
  • activeColor -激活状态选项卡的标签和图标颜色。
  • inactiveColor - 非激活状态选项卡的标签和图标颜色。
  • barStyle-底部导航栏的样式。 如果你在Android上具有半透明的导航栏，则可以在此处设置底部填充：barStyle = {{paddingBottom: 48}}。
  • initialRouteName -第一次加载时初始选项卡路由的 routeName。
  • order -定义选项卡顺序的 routeNames 数组。
  • paths - 提供 routeName 到 path 配置的映射, 它重写 routeConfigs 中设置的路径。
  • backBehavior-initialRoute返回初始选项卡，order返回上一个选项卡，history返回上次访问标签或none。

例子:

export default createMaterialBottomTabNavigator(
  {
    Album: { screen: Album },
    Library: { screen: Library },
    History: { screen: History },
    Cart: { screen: Cart },
  },
  {
    initialRouteName: 'Album',
    activeColor: '#f0edf6',
    inactiveColor: '#3e2465',
    barStyle: { backgroundColor: '#694fad' },
  }
);

用于导航器内部页面的navigationOptions

标题

可用作headerTitle和tabBarLabel的后备的通用标题。

tabBarIcon

React 元素或给定{ focused: boolean, horizontal: boolean, tintColor: string }返回一个 React.Node的函数，用于在标签栏中显示。 当设备处于横屏时，horizontal 是 true；当设备处于竖屏时false。 每当设备方向发生变化时, 都会重新渲染该图标。

tabBarColor

当屏幕对应的标签处于活动状态时，标签栏的颜色。 用于波纹效果。 仅当shifting为true时才支持。

tabBarLabel

在选项卡栏中显示的选项卡标题字符串。 未定义时，使用场景title。 要隐藏，请参阅上一节中的labeled选项。

tabBarBadge

Badge to show on the tab icon, can be true to show a dot, string or number to show text.

tabBarAccessibilityLabel

选项卡按钮的无障碍标签。 当用户点击该选项卡时，该标签会被屏幕阅读器阅读（用于方便视障人士使用 手机，鼓励有条件的开发者设置该标签）， 如果您的选项卡的没有标签, 强烈建议您设置此选项。

tabBarTestID

用于在测试中找到该选项卡按钮的 ID。

tabBarOnPress

处理点击事件的回调; 该参数是一个对象，其中包含：

  • navigation：页面的 navigation props
  • defaultHandler: tab press 的默认 handler

在开始转换到下一个场景之前添加自定义逻辑（点击的场景）。 设置tabBarOnPress时，需要调用defaultHandler来执行默认操作（即switch选项卡）。

使用 react-native-paper （可选）

你可以使用支持主题的react-native-paper 来自定义主题，方法是 使用 react-native-paper 的 Provider 包裹整个 App 一种常用的方法是当App 使用深色主题时，自定义页面的背景色。 请阅读 instructions on react-native-paper使用文档来学习如何自定义主题。
