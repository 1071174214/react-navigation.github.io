iPhone X support

By default React Navigation aids in ensuring your application displays correctly on the iPhoneX. 它通过在可能与传感器集群（“齐刘海”）或 Home 主页指示器交互的UI元素内部使用SafeAreaView来实现。

隐藏/自定义导航栏或选项卡栏

However, if you're overriding the default navigation bar it's important to ensure your UI doesn't interfere with either of those hardware elements.

For example, if I render nothing for the header or tabBarComponent nothing renders

const Tabs = TabNavigator({
  ...
}, {
  tabBarComponent: () => null,
});

export default StackNavigator({
  ...
}, {
  headerMode: 'none',
});

To fix this issue you can wrap your content in a SafeAreaView, which can be imported from either react-native or react-navigation.

import { SafeAreaView } from 'react-navigation';

class App extends Component {
  render() {
    return (
      <SafeAreaView style={styles.container}>
        <Text style={styles.paragraph}>
          This is top text.
        </Text>
        <Text style={styles.paragraph}>
          This is bottom text.
        </Text>
      </SafeAreaView>
    );
  }
}

这将检测应用程序是否在 iPhone X 上运行，如果是，确保内容不会被任何硬件元素遮挡。

Landscape Mode（横屏模式）

即使你的应用程序在横屏模式下使用默认的 navigation bar 和 tab bar，确保内容不会被齐刘海遮挡也很重要。

要解决此问题，您可以再次将您的内容包装在SafeAreaView中。 这不会与竖屏模式下的 navigation bar 和 tab bar 的默认行为相冲突。

总之，请在使用 React Navigation 导航器注册的页面上使用SafeAreaView组件。
