Community-developed Navigators and Libraries

Navigators

过度动画

Fluid Transitions是一个库，它在使用 react-navigation 的页面之间导航期间提供共享元素转换。

共享元素转换是在导航转换过程中，将一个屏幕中的元素转换为另一个屏幕中的相应元素的可视化。

该库实现了一个称为FluidNavigator的自定义导航器，它实现了所有这些功能和更多其它的可能。

链接

github.com/fram-x/FluidTransitions

Libraries

react-navigation-collapsible

react-navigation-collapsible 是一个高阶组件库，它通过设置 navigationOptions 使得页面的 Header 可折叠。

由于 react-navigation 的 Header 被设计为 Animated 组件。 您可以通过从 ScrollView 或 FlatList 向 header 传递Animated.Value，从而使 header 可以使用动画效果

链接

github.com/benevbright/react-navigation-collapsible

Snack 上的示例

react-native-screens

这个项目旨在暴露原生的导航容器给 React Native 和 React Navigation，可以在v2.14.0之前集成它。使用 react-native-screens可以带来如下几个好处：支持 iOS 的 "reachability feature" 、优化双端的内存消耗。

链接

github.com/kmagiera/react-native-screens

react-navigation-header-buttons

帮助您渲染导航栏中的按钮并处理样式 它试图模仿原生的导航栏按钮的外观, 并尝试提供一个简单的界面, 供您与之交互。

链接

github.com/vonovak/react-navigation-header-buttons

Expo 上的示例

react-navigation-props-mapper

提供简单的 hoc, 直接将 react-navigation 的prop 映射到页面组件， 而不是使用const user = this.props.navigation.getParam(activeUser, null)，只要const user = this.props.activeUser这样调用就行了

链接

github.com/vonovak/react-navigation-props-mapper
