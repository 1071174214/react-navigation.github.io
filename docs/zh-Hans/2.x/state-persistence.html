State persistence

你可能希望将用户的位置保存在应用程序中, 以便在应用程序重新启动后立即返回到同一位置。

这在开发过程中特别有用，因为它允许开发人员在刷新应用程序时保持在同一页面上。

注意：此功能目前被认为是实验性的，因为本页末尾列出了警告。 谨慎使用！

用法

你可以通过使用persistenceKey来为你的顶级导航器启用持久化：

const AppNavigator = createStackNavigator({...});

const App = () => <AppNavigator persistenceKey={"NavigationState"} />;

这个提供的 key 将用作响应原生AsyncStorage的键以保存 JSON 导航状态。

开发模式

此功能在开发模式中特别有用。你可以使用以下方法有选择地启用它:

const AppNavigator = createStackNavigator({...});
const navigationPersistenceKey = __DEV__ ? "NavigationStateDEV" : null;
const App = () => <AppNavigator persistenceKey={navigationPersistenceKey} />;

加载 View

因为 state 是异步持久的，所以当AsyncStorage请求完成时，应用程序必须呈现一个空/正在加载的视图。 要自定义在此时间内呈现的加载视图, 可以使用 renderLoadingExperimental prop:

<AppNavigator
  persistenceKey={...}
  renderLoadingExperimental={() => <ActivityIndicator />}
/>

注意：此API可能在将来发生变化，这就是为什么它被标记为实验性的原因

警告：可序列化的 State

每个 param、route和 navigation state 必须完全 JSON 序列化才能使此功能正常工作。 这意味着你的 route 和 param 必须不包含函数、类实例或递归数据结构。

警告：Route/Router 定义更改

当你的应用程序代码更改为在导航状态下支持给定路线的新路线或不同路线时，应用程序可能会在呈现旧导航状态时中断。

在你重新配置路线和导航层次结构时，这可能会在开发过程中定期发生。 但是当你发布你的应用程序的新版本时，它也可能发生在生产中！

保守的行为是在应用程序更新后擦除导航状态。 最简单的方法是为你发布给用户的每个版本引用不同的持久性密钥。

React Navigation 使用 React 的componentDidCatch功能试图减轻由路由定义更改引起的崩溃，但这被认为是实验性的，可能无法捕获所有错误。
