createDrawerNavigator

要使用此导航器，请确保已安装 react-navigation及其依赖项，然后安装<a href =“ https://github.com/react- navigation/tabs“> react-navigation-tabs 。

yarn add react-navigation-drawer

API

import { createDrawerNavigator } from 'react-navigation-drawer';

createDrawerNavigator(RouteConfigs, DrawerNavigatorConfig);

RouteConfigs

路由配置对象是从路由名称到路由配置的映射，它告诉导航器该路由要呈现的内容，请参阅createStackNavigator中的示例 。

DrawerNavigatorConfig

  • drawerBackgroundColor - 使用抽屉背景色， 默认值：white。

  • drawerPosition - 可选值： left 或 right， 默认值： left。

  • drawerType - drawer的类型. 它确定抽屉的外观和动画效果。

    • front: 传统的抽屉覆盖屏幕，并在屏幕后面覆盖。
    • back: 滑动后，抽屉就会显示在屏幕后面。
    • slide: 屏幕和抽屉都在滑动时滑动以露出抽屉。

  • drawerWidth - 数字或返回抽屉宽度的函数。 如果提供了功能，则屏幕尺寸更改时将再次调用该功能。

  • edgeWidth - 允许定义滑动手势应激活的距离内容视图边缘的距离

  • hideStatusBar - 设置为true时，每当抽屉被拉出或处于“打开”状态时，抽屉组件将隐藏OS状态栏。

  • statusBarAnimation - 隐藏状态栏时的动画。 与hideStatusBar结合使用。

  • keyboardDismissMode - 开始滑动手势时是否应该关闭键盘。 默认为'on-drag'。 设置为'none'可以禁用键盘处理。

  • minSwipeDistance - 应激活打开抽屉的最小滑动距离阈值。

  • overlayColor - 打开抽屉时，颜色叠加层将显示在内容视图的顶部。 抽屉打开时，不透明度从0变为1。

  • gestureHandlerProps - 传递给基础平移手势处理程序的道具。

  • lazy - 屏幕是否应在首次访问时呈现。 默认为true。 如果要在初始渲染时渲染所有屏幕，请将其设置为false。

  • unmountInactiveRoutes-在离开屏幕时是否应卸载屏幕。 默认值： false。

  • contentComponent -用于呈现抽屉内容 (例如, 导航项) 的组件。 接收抽屉的navigation道具和drawerOpenProgress。 默认为 DrawerItems。 有关详细信息, 请参阅下文。

  • contentOptions -配置抽屉内容, 请参阅下面。

  • navigationOptions-导航器本身的导航选项，用于配置父导航器

  • defaultNavigationOptions - 用于屏幕的默认导航选项

几个被传递到底层路由以修改导航逻辑的选项：

  • initialRouteName -第一次加载时初始选项卡路由的 routeName。
  • order -定义选项卡顺序的 routeNames 数组。
  • paths - 提供 routeName 到 path 配置的映射, 它重写 routeConfigs 中设置的路径。
  • backBehavior - 控制 "返回" 按钮是否会导致 Tab 页切换到初始 Tab 页? 如果是, 设置为 initialRoute, 否则 none。 默认为 initialRoute的行为。

提供一个自定义的contentComponent

抽屉的默认组件是可滚动的，只包含 RouteConfig 中路由的链接。 你可以轻松地覆盖默认组件，以向抽屉中添加页眉，页脚或其他内容。 默认情况下，抽屉可滚动并支持 iPhone X 安全区域。 如果您自定义内容，请务必将内容包装在 SafeAreaView 中：

import SafeAreaView from 'react-native-safe-area-view';
import { DrawerItems } from 'react-navigation-drawer';

const CustomDrawerContentComponent = props => (
  <ScrollView>
    <SafeAreaView
      style={styles.container}
      forceInset={{ top: 'always', horizontal: 'never' }}
    >
      <DrawerItems {...props} />
    </SafeAreaView>
  </ScrollView>
);

const styles = StyleSheet.create({
  container: {
    flex: 1,
  },
});

contentComponent还收到了一个名为drawerOpenProgress的道具，该道具是一个Reanimated Node，它表示抽屉的动画位置（0为关闭； 1为打开）。 这使您可以在contentComponent中进行有趣的动画，例如抽屉内容的视差运动：

const CustomDrawerContentComponent = props => {
  const translateX = props.drawerOpenProgress.interpolate({
    inputRange: [0, 1],
    outputRange: [-100, 0],
  });

  return (
    <Animated.View style={{ transform: [{ translateX }] }}>
      {/* ... drawer contents */}
    </Animated.View>
  );
};

DrawerItems的contentOptions

  • items - 路由数组，可以修改或覆盖
  • activeItemKey - 识别活动路线的关键字
  • activeTintColor -活动选项卡的标签和图标颜色。
  • activeBackgroundColor -活动选项卡的背景色。
  • inactiveTintColor -"非活动" 选项卡的标签和图标颜色。
  • inactiveBackgroundColor -非活动选项卡的背景色。
  • onItemPress({ route, focused }) - 按下项目时要调用的功能
  • itemsContainerStyle -内容节的样式对象
  • itemStyle 样式对象的单个项, 可以包含图标和/或标签
  • labelStyle 样式对象要覆盖 文本 样式内部内容部分, 当您的标签是一个字符串
  • activeLabelStyle 样式对象若要改写活动标签的 文本 样式, 则标签为字符串 (合并使用 labelStyle)
  • inactiveLabelStyle 样式对象在标签为字符串时覆盖 文本 样式的非活动标签 (与 labelStyle 合并)
  • iconContainerStyle - 用于覆盖View图标容器样式的样式对象。

例子:

contentOptions: {
  activeTintColor: '#e91e63',
  itemsContainerStyle: {
    marginVertical: 0,
  },
  iconContainerStyle: {
    opacity: 1
  }
}

页面导航选项

标题

可用作headerTitle和tabBarLabel的后备的通用标题。

drawerLabel

字符串，React 元素或给定{ focused: boolean, tintColor: string }返回一个 React.Node 的函数，用于在标签栏中显示。 未定义时，使用title。

drawerIcon

React 元素或给定{ focused: boolean, tintColor: string }的函数返回一个 React.Node，用于在标签栏中显示。

drawerLockMode

指定抽屉的锁定模式。 这也可以通过在顶级路由器上使用 screenProps.drawerLockMode 动态更新。

将抽屉导航器嵌套在其它导航的内部

如果抽屉导航器嵌套在另一个提供某个UI的导航器中，例如选项卡导航器或堆栈导航器，则抽屉将从这些导航器的UI下呈现。 抽屉会出现在标签栏的下方并位于堆栈的标题下方。 你将需要将抽屉导航器作为抽屉应该呈现在其UI之上的任何导航器的父级。
